# Устранение неполадок MED-V


В этом разделе приведены сведения, которые помогут вам устранить общие проблемы с виртуализацией классической версии Microsoft Enterprise (MED-V).

## Изменение разрешения узла и максимизация рабочей области MED-V приводит к тому, что настольный компьютер будет выглядеть черным


При работе в режиме полного рабочего стола, если изменить разрешение узла и затем развернуть окно рабочей области для MED-V, Рабочий стол будет черным, а рабочее пространство MED-v может не ответить.

### Решение

Остановите и запустите рабочую область MED-V.

## Запуск рабочей области MED-V с отключенным сетевым адаптером и последующее включение адаптера не восстанавливает сетевое подключение


Если вы настроили рабочую область MED-V в режиме моста и затем запускаете рабочую область MED-V, когда сетевой адаптер отключен, подключение к сети с помощью этого адаптера будет восстановлено.

### Решение

Остановите и запустите рабочую область MED-V.

## Изображение может использоваться только одним пользователем Windows на компьютере.


Образ рабочей области MED-V может использоваться только тем пользователем Windows, который загрузил или импортировал изображение. Этот пользователь не является администратором, имеющим разрешения на доступ к папке, в которой находятся Скачанные изображения.

### Решение

Вручную измените список управления доступом (ACL) в хранилище изображений.

## При установке MED-V с помощью Configuration Manager с включенными правами пользователей не удается выполнить удаление


Если для установки MED-V используется Microsoft System Center Configuration Manager и режим выполнения пакета установлен в значение права пользователей, при удалении из него появляется сообщение об ошибке, в котором говорится, что только административные пользователи могут удалить MED-V.

### Решение

При создании пакета Configuration Manager для MED-V настройте режим выполнения для прав администратора.

## При установке MED-V с помощью корпоративной системы развертывания, где установка настроена для запуска клиента после установки, вы не сможете запустить клиент


Если для установки MED-V используется корпоративная система развертывания, а установочный пакет настроен на запуск клиента MED-V после установки, то после его запуска с помощью системной учетной записи вы не увидите, что клиент запущен (за исключением области уведомлений), и вы не можете взаимодействовать с ним.

### Решение

При установке MED-V с помощью корпоративной системы развертывания используйте параметр *Start \ _MEDV = 0* . msi.

## Не запускается тестовый образ для MED-V


Если изображение теста MED-V не запускается, оно не будет восстановлено, а все последующие запуска будут завершаться сбоем с сообщением об ошибке "GINA не удалось загрузить".

### Решение

Удалите существующее тестовое изображение и повторно создайте его.

## При попытке присоединиться к домену с неверными учетными данными изображение не будет успешно присоединяться к домену.


Если при попытке присоединиться к домену в стандартном блоке "присоединиться к домену" произошла ошибка конфигурации, которая является частью сценария настройки при первом запуске виртуальной машины, это приводит к сбою рабочей области MED-V. После восстановления ошибки конфигурации образ, включенный в рабочую область MED-V, не может присоединиться к домену.

### Решение

Если вы развернули изображение, перераспределите его. Если изображением является тестовое изображение, повторно создайте изображение.

## MED-V не поддерживает несколько мониторов


MED-V не поддерживает отображение опубликованных приложений на нескольких мониторах. Опубликованные приложения и другие клиентские окна могут отображаться на неправильном экране, а иногда после отключения экрана приложение MED-V пытается отправить экран на монитор, чтобы подключенный монитор отображался пустым.

### Решение

Отключите дополнительный экран и перезапустите клиент.

## Рабочая область MED-V может не запуститься, если при запуске MED-V происходит сбой узла.


Если при запуске рабочей области MED-V происходит сбой узла и появляется сообщение об ошибке "корневой элемент отсутствует", Рабочая область MED-V может добавлять данные в пустой файл конфигурации виртуальных машин (VMC), что приводит к сбою процесса запуска.

### Решение

Замените пустой файл VMC файлом VMC из базового образа.

## Клавиатура не отвечает в опубликованных окнах приложений


Если в рабочей области MED-V вы нажмете клавишу с логотипом Windows, то клавиатура больше не отвечает на опубликованные окна приложений.

### Решение

Нажмите клавишу с логотипом Windows, пока приложение находится в фокусе.

## Доменная область MED-V не обновляет учетные данные домена.


При использовании постоянной рабочей области MED-V в среде домена при изменении пароля домена клиент MED-V не обновляет учетные данные домена рабочей области MED-V. Когда опубликованное приложение пытается получить доступ к сетевому ресурсу, появится сообщение об ошибке с уведомлением о том, что срок действия ваших учетных данных истек.

### Решение

Перезапустите операционную систему MED-V Workspace.

## Развернутые окна опубликованных приложений закрывают панель задач Hosting


Если окно опубликованного приложения развернуто на весь экран, оно может закрываются на панели задач Host.

### Решение

Выполните одно из следующих действий.

Сверните окно опубликованного приложения, чтобы получить доступ к области уведомлений, и перезапустите рабочую область MED-V.

Сверните окно опубликованного приложения, а затем восстановите его в развернутом состоянии.

## Не работает Добавление пользователей или групп в диспетчере конфигурации MED-V Server


При добавлении пользователей или групп в диалоговом окне **Выбор пользователей или** групп выбранные пользователи или группы не добавляются в список управления доступом в диспетчере конфигураций MED-V Server.

### Решение

Добавьте пользователей и группы с помощью диалогового окна " **Введите имена пользователей или групп** ". Подробные сведения о том, [как установить и настроить серверный компонент MED-V, приведены в этой](how-to-install-and-configure-the-med-v-server-component.md#bkmk-configuringpermissions)странице.

## Приложение MED-V не работает на компьютерах с Windows Virtual PC для Windows 7


Для работы MED-V требуется Windows Virtual PC2007. Windows Virtual PC для Windows7 и Virtual PC2007 SP1 нельзя установить на одном компьютере.

### Решение

Удалите Virtual PC для Windows7 перед установкой виртуальных PC2007 SP1 и MED-V.

## <a href="" id="med-v-does-not-support-virtual-pc-and-windows-xp-mode-images-"></a>MED-V не поддерживает виртуальные компьютеры и режимы режима WindowsXP


Приложение MED-V 1.0 SP1 не поддерживает образы, созданные Windows Virtual PC для Windows7. При использовании Virtual PC для Windows7 Image происходит сбой клиента во время запуска.

### Решение

Создавайте образы MED-V с помощью Virtual PC2007 SP1.

## Брандмауэр Windows блокирует виртуальную сетевую активность PC2007 SP1


По умолчанию брандмауэр Windows блокирует активность виртуальных PC2007 пакета обновления 1 (SP1) и, когда виртуальный PC2007 SP1 инициируется на клиентском компьютере, есть сообщение брандмауэра, блокирующее последовательность запуска и весь доступ к сети.

### Решение

Обновите исключение брандмауэра с помощью групповой политики, прежде чем MED-V используется конечным пользователем.

## При обновлении клиента появляется сообщение об ошибке


При обновлении клиента из MED-V 1.0 на MED-V 1.0 с пакетом обновления 1 (SP1) может появиться сообщение о том, что Рабочая область MED-V не определена.

### Решение

Закройте клиент и перезапустите его.
