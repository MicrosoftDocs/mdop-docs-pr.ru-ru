# Создание образа Virtual PC для MED-V


Чтобы создать Virtual PC (VPC-образ) для MED – V, необходимо выполнить указанные ниже действия.

1.  [Создайте образ VPC](#bkmk-creatingavirtualmachinebyusingmicrosoftvirtualpc).

2.  [Установите пакет. msi для MED-V Workspace на образ VPC](#bkmk-howtoinstallthemedvworkspacemsipackage).

3.  [Запустите средство предварительных требований для виртуальной машины MED-V на образе VPC](#bkmk-howtorunthevirtualmachineprerequisitestool).

4.  [Настройте вручную необходимые виртуальные машины для образа VPC](#bkmk-howtoconfiguremedvvirtualmachinemanualinstallationprerequisites).

5.  [Настройте Sysprep для изображений MED-V](#bkmk-howtoconfiguresysprepformedvimages) (необязательно).

6.  [Отключите Microsoft Virtual PC](#bkmk-turningoffmicrosoftvirtualpc).

## <a href="" id="bkmk-creatingavirtualmachinebyusingmicrosoftvirtualpc"></a>Создание образа Virtual PC с помощью виртуального компьютера Microsoft Virtual PC


Чтобы создать образ виртуального компьютера с помощью Microsoft Virtual PC, ознакомьтесь с документацией по Virtual PC.

Дополнительные сведения см. в следующих разделах:

-   [Справка по Windows Virtual PC](https://go.microsoft.com/fwlink/?LinkId=182378)

-   [Создание виртуальной машины и установка гостевой операционной системы](https://go.microsoft.com/fwlink/?LinkId=182379)

## <a href="" id="bkmk-howtoinstallthemedvworkspacemsipackage"></a>Установка пакета "MED-V Workspace" (пакет MSI)


После создания образа Virtual PC установите на него пакет Windows MED-V Workspace. msi.

**Установка образа рабочей области для MED-V**

1.  Запустите виртуальную машину и скопируйте пакет MED-V Workspace из рабочей области для Windows Inside.

    Пакет для рабочей области MED-V называется *MED-V\_workspace\_x.msi*, где *x* — номер версии.

    Например, *MED-V\_workspace\_1.0.65.msi*.

2.  Дважды щелкните пакет Office MED-V Workspace и следуйте инструкциям мастера установки.

    **Примечание.**  
    Если после выпуска новой версии MED-V вы обновите существующий виртуальный компьютер, удалите его, а затем перезагрузите компьютер и установите новый пакет для MED-V Workspace. msi.



~~~
**Note**  
After the MED-V workspace .msi package is installed, other products that replace GINA cannot be installed.
~~~



## <a href="" id="bkmk-howtorunthevirtualmachineprerequisitestool"></a>Выполнение средства предварительных требований для виртуальной машины


Предварительные требования виртуальной машины (VM) — это мастер, который автоматизирует некоторые предварительные условия.

**Примечание.**  
Несмотря на то, что в мастере можно настроить многие параметры, свойства, необходимые для правильной работы MED-V, не настраиваются.



**Запуск средства предварительных требований для виртуальной машины**

1.  После установки пакета MED-V Workspace. msi в меню **Пуск** выберите пункт **все программы, необходимые для работы &gt; с &gt; предварительными требованиями к виртуальной машине для MED-v**.

    **Примечание.**  
    Пользователь, который запускает средство предварительной установки виртуальной машины, должен иметь права локального администратора и должен быть единственным входным пользователем.



~~~
The **MED-V VM Prerequisite Wizard Welcome** page appears.
~~~

2. Нажмите кнопку **Далее**.

3. На странице " **Параметры Windows** " из указанных ниже настраиваемых свойств выберите нужные настройки.

   -   **Очистка личных данных пользователей**

   -   **Очистка папки Temp Local profiles**

   -   **Отключение звуков на следующих событиях Windows: запуск, вход, выход из системы**

   **Примечание.**  
   Не включайте функцию "Заставка" на странице Windows в групповой политике.



4. Нажмите кнопку **Далее**.

5. На странице **параметры Internet Explorer** в указанных ниже настраиваемых свойствах выберите нужные настройки.

   -   **Не используйте функцию автоматического завершения**

   -   **Отключение повторного использования окон при запуске сочетаний клавиш**

   -   **Очистка журнала браузера**

   -   **Включение обзора с вкладками в Internet Explorer 7**

6. Нажмите кнопку **Далее**.

7. На странице " **службы Windows** " выберите нужные настройки из следующих настраиваемых свойств:

   -   **Служба центра обеспечения безопасности**

   -   **Служба планировщика заданий**

   -   **Служба автоматических обновлений**

   -   **Служба восстановления системы**

   -   **Служба индексирования**

   -   **Беспроводная настройка**

   -   **Быстрая совместимость с переключением пользователей**

8. Нажмите кнопку **Далее**.

9. На странице **автоматического входа в Windows** выполните указанные ниже действия.

   1.  Установите флажок **включить автоматический вход в Windows** .

   2.  Назначение **имени пользователя** и **пароля**.

10. Нажмите кнопку **Применить**, а затем в появившемся диалоговом окне подтверждения нажмите кнопку **Да**.

11. На странице " **Сводка** " нажмите кнопку **"Готово"** , чтобы завершить работу мастера.

**Примечание.**  
Убедитесь, что групповые политики не перезаписывают обязательные параметры, заданные в инструменте "предварительные требования".



## <a href="" id="bkmk-howtoconfiguremedvvirtualmachinemanualinstallationprerequisites"></a>Настройка предварительных требований для установки виртуальной машины на основе MED-V


Некоторые из конфигураций нельзя настроить с помощью средства предварительных требований виртуальных машин, которые необходимо выполнить вручную.

-   Параметры виртуальной машины

    Рекомендуется настроить следующие параметры виртуальной машины в консоли Microsoft Virtual PC:

    -   Отключите дисководы для гибких дисков.

    -   Отключите функцию отмены — диски **( &gt; Отмена параметров — диски**).

    -   Убедитесь, что на изображении только один виртуальный ЦП.

    -   Устраните взаимодействие между виртуальной машиной и пользователем, где они не связаны с опубликованными приложениями (например, сообщениями, требующими ввода данных пользователем).

-   Параметры изображения

    Настройте следующие параметры вручную в изображении:

    1.  В окне **Свойства: параметры электропитание** отключите режим сна и спящий режим.

    2.  Установка последних обновлений для Windows.

    3.  В диалоговом окне **Загрузка и восстановление Windows** в разделе **системный сбой** снимите флажок **автоматически перезапустить** .

    4.  Убедитесь, что на изображении используется ключ лицензии VLK.

-   Установка дополнений VPC

    В меню **действия** выберите **установить или обновить дополнения для виртуальной машины**.

-   Настройка печати

    Вы можете настроить печать из рабочей области MED-V одним из указанных ниже способов.

    -   Добавьте принтер на виртуальную машину.

    -   Разрешить печать с использованием принтеров, настроенных на главном компьютере.

## <a href="" id="bkmk-howtoconfiguresysprepformedvimages"></a>Настройка Sysprep для изображений MED-V


В рабочей области MED-V можно настроить Sysprep для присвоения уникального идентификатора безопасности (SID), особенно в том случае, если на одном компьютере работают несколько рабочих областей MED-V. Использовать Sysprep для присоединения к домену не рекомендуется. Вместо этого используйте действие сценария присоединиться к домену MED-V, как описано в разделе [Настройка действий сценария](how-to-set-up-script-actions.md).

**Примечание.**  
Sysprep — это программа подготовки системы Microsoft для операционной системы Windows.



**Настройка Sysprep в рабочей области MED-V**

1.  Создайте каталог в корневом каталоге системного диска с именем *Sysprep*.

2.  С установочного компакт-диска Windows извлеките *deploy.cab* в корневой каталог системного диска или скачайте последнюю версию средств развертывания на веб-сайте Майкрософт.

    -   Для Windows 2000 в разделе [Обновление средств развертывания для windows 2000](https://go.microsoft.com/fwlink/?LinkId=143001).

    -   В Windows XP можно найти в разделе [Обновление средств развертывания для Windows XP](https://go.microsoft.com/fwlink/?LinkId=143000).

3.  Запустите **Диспетчер установки** (setupmgr.exe).

4.  Следуйте указаниям мастера настройки диспетчера установки.

После настройки Sysprep и создания рабочей области для MED-V необходимо выполнить Sysprep.

**Запуск программы Sysprep**

1.  В папке Sysprep, расположенной в корневом каталоге системного диска, запустите средство подготовки системы (Sysprep.exe).

2.  В появившемся окне с предупреждением нажмите кнопку **ОК**.

3.  В диалоговом окне **Параметры Sysprep** установите флажок **не сбрасывать льготный период для активации** и **Используйте мини-установку** .

4.  Нажмите кнопку " **запечатать**".

5.  Если вы не удовлетворены данными, указанными в появившемся диалоговом окне подтверждения, нажмите кнопку **Отмена** и измените параметры.

6.  Нажмите кнопку **ОК** , чтобы завершить процесс подготовки системы.

## <a href="" id="bkmk-turningoffmicrosoftvirtualpc"></a>Отключение Microsoft Virtual PC


После того как все компоненты будут установлены и настроены, закройте Virtual PC (Майкрософт) и выберите **Отключить**.
