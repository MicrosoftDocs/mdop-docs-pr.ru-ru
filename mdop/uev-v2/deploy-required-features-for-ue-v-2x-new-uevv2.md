# Развертывание необходимых компонентов для UE-v 2.x


Для всех развертываний Microsoft User Experience Virtualization (UE-V) 2,0, 2,1 и 2,1 SP1 нужны следующие функции:

-   [Развертывание места хранения параметров](#ssl) , доступного для конечных пользователей.

    Это стандартная сетевая общей папке, в которой хранятся и извлекаются параметры пользователя.

-   [Выбор метода конфигурации для UE-V](#config)

    UE-V можно разворачивать и настраивать с помощью стандартных средств управления, включая групповую политику, Диспетчер конфигураций, а также инфраструктуру управления Windows и PowerShell.

-   [Развертывание агента UE-V](#agent) для установки на каждом компьютере, который синхронизирует параметры.

    Это отслеживает зарегистрированные приложения и операционную систему на предмет изменений параметров и синхронизирует эти параметры между компьютерами.

В этой статье описано, как развертывать эти функции.

## <a href="" id="ssl"></a>Развертывание расположения хранилища параметров UE-V


Для UE-V требуется расположение, в котором нужно хранить параметры пользователя в файлах пакета параметров. Вы можете настроить место хранения параметров одним из указанных ниже способов.

-   Создание собственного места для хранения параметров

-   Использование существующей службы каталогов Active Directory для хранения параметров

Если вы не создаете место хранения параметров, агент UE-V Agent будет по умолчанию использовать Active Directory (AD).

**Примечание.**  
В [плане производительности и емкости](https://technet.microsoft.com/library/dn458932.aspx#capacity) , а также для снижения проблем с задержкой сети, создайте места хранения параметров в тех же локальных сетях, где находятся компьютеры пользователей. Рекомендуем использовать для хранения параметров 20 МБ свободного места на диске для каждого пользователя.



### Создание места хранения параметров UE-V

Перед определением места хранения параметров необходимо создать корневой каталог с разрешениями на чтение и запись для пользователей, которые хранят параметры в общем доступе. Агент UE-V создает пользовательские папки под этим корневым каталогом.

Место хранения параметров определяется путем настройки параметра конфигурации SettingsStoragePath, который можно настроить с помощью одного из следующих методов:

-   При [развертывании агента UE-V](#agent) с помощью параметра командной строки или пакетного сценария

-   С помощью параметров [групповой политики](https://technet.microsoft.com/library/dn458893.aspx)

-   С помощью [пакета конфигурации System Center](https://technet.microsoft.com/library/dn458917.aspx) для UE-V

-   После установки агента UE-V с помощью [Windows PowerShell или инструментария управления Windows (WMI)](https://technet.microsoft.com/library/dn458937.aspx)

Путь должен быть указан в формате UNC сервера и общего доступа. Например, **\\\\Server\\Settingsshare\\**. Этот параметр конфигурации поддерживает использование переменных для включения конкретных сценариев синхронизации. Например, вы можете использовать `%username%\%computername%` переменные для сохранения параметров конечного пользователя в следующих сценариях:

-   Завершение работы пользователей, использующих несколько физических компьютеров предприятия

-   Корпоративные компьютеры, используемые несколькими конечными пользователями

Агент UE-V динамически создает путь к хранилищу параметров для определенного пользователя, с именем скрытой системной папки в `SettingsPackages` соответствии с параметром конфигурации **SettingsStoragePath**. Агент считывает и записывает в это расположение параметры, заданные зарегистрированными шаблонами расположений параметров UE-V.

**Параметры UE-V определяются правилом "Последнее запись WINS".** Если место хранения параметров одинаково для пользователей с несколькими управляемыми компьютерами, один агент UE-V производит чтение и запись в расположение параметров независимо от агентов, запущенных на других компьютерах. Последние записанные параметры и значения — это те, которые применяются при чтении следующего агента из места хранения параметров.

**Развертывание места хранения параметров.** Выполните указанные ниже действия, чтобы определить место хранения параметров, а не использование существующей службы Active Directory. 2.  Установите для раздела реестра значение *1*.



### Использование службы каталогов Active Directory с UE-V 2. x

Агент UE-V использует Active Directory (AD) по умолчанию, если место хранения параметров не определено в противном случае. В этих случаях агент UE-V динамически создает папку хранения параметров в корне основного каталога рекламных объявлений каждого пользователя. Но если настраиваемый параметр каталога настроен в AD, вместо него используется этот каталог.

## <a href="" id="config"></a>Выберите метод конфигурации для UE-V 2. x


Вы хотите узнать, какой метод конфигурации вы используете для управления UE-V после развертывания, так как это будет метод конфигурации, который вы используете для развертывания UE-V Agent. Обычно это метод конфигурации, который уже используется в среде, например Windows PowerShell или Configuration Manager.

Агенты UE-V до, во время и после установки агента UE-V можно настроить в зависимости от используемого метода конфигурации.

-   [Групповая политика](https://technet.microsoft.com/library/dn458893.aspx)**:** вы можете использовать существующую инфраструктуру групповой политики для настройки UE-v до или после развертывания агента UE-v. Шаблон файлов ADMX групповой политики UE-V обеспечивает централизованное управление общими параметрами конфигурации агента UE-V и включает параметры для настройки синхронизации UE-V.

    **Установка шаблонов ADMX для групповой политики UE-V:** Шаблоны ADMX из групповой политики для UE-v настройте параметры синхронизации для агента UE-2.2 и разрешите централизованное управление общими параметрами конфигурации агента UE-V с помощью существующей инфраструктуры групповой политики.

    Ниже перечислены поддерживаемые операционные системы для контроллера домена, на котором развертываются объекты групповой политики.

    Windows Server2008R2

    Windows Server 2012 и Windows Server 2012 R2

-   [Диспетчер конфигураций](https://technet.microsoft.com/library/dn458917.aspx)**:** пакет конфигурации UE-V позволяет использовать параметры соответствия требованиям компонента System Center Configuration Manager 2012 или более поздней версии для применения согласованных конфигураций на сайтах, где установлены UE-V и Configuration Manager.

-   [Windows PowerShell и WMI](https://technet.microsoft.com/library/dn458937.aspx)**:** для изменения конфигураций после установки агента UE-V можно использовать команды сценария для Windows PowerShell и инструментария управления Windows (WMI).

    **Примечание.**  
    Изменение реестра может привести к потере данных или перестает отвечать на запросы компьютера. Мы рекомендуем использовать другие методы настройки.



-   **Установка из командной строки или пакетного сценария:** Параметры, которые используются при [развертывании агента UE-v](#agent) , настройте множество параметров ue-v. Электронные системы распространения программного обеспечения, такие как System Center 2012 Configuration Manager, используют эти параметры для настройки клиентов при развертывании и установке программного обеспечения агента UE-V.

## <a href="" id="agent"></a>Развертывание агента UE-V 2. x


Агент UE-V является ядром развертывания UE-v и должен выполняться на каждом компьютере, использующем UE-V для синхронизации приложений и параметров Windows.

**Установочные файлы агента UE-V:** Один установочный файл, AgentSetup.exe, устанавливает агент UE-V Agent как в 32-, так и в 64-разрядной операционной системе. Кроме того, предоставляются AgentSetupx86.msiные и AgentSetupx64.msi специфические файлы установщика Windows, а так как они меньше, они могут ускорить развертывание агента. [Параметры командной строки для установщика AgentSetup.exe](#params) поддерживаются и для установки установщика Windows.

**Важно.**  
Во время установки или удаления агента UE-V вы можете использовать либо файл AgentSetup.exe, либо &lt; файл Arch AgentSetup &gt; . msi, но не оба варианта. Для удаления агента UE-V, который использовался для установки UE-V Agent, необходимо использовать один и тот же файл.



### Развертывание агента UE-V

Для развертывания агента UE-V можно использовать следующие методы:

-   Система решений для электронного распространения программного обеспечения (ESD), например Configuration Manager, которая может установить файл установщика Windows (MSI).

-   Сценарий установки, который ссылается на файл установщика Windows (MSI), хранящийся централизованно в общем доступе.

-   Программа установки, запускаемая вручную на компьютере.

Выполните описанные ниже действия, чтобы развернуть агент UE-V из сетевого общего доступа.

**Установка и настройка агента UE-V для сетевого общего доступа**

### Обновление агента UE-V

Обновления программного обеспечения агента UE-V предоставляются с помощью центра обновления Майкрософт. Обновления агента UE-V можно развертывать с помощью систем инфраструктуры для корпоративного распространения программного обеспечения (ESD).

Во время обновления агента UE-V можно обновить стандартную группу шаблонов расположений параметров для приложений Майкрософт и параметров Windows.

### Обновление агента UE-V 2. x

Агент UE-V 2. x предлагает множество новых функций и изменяет способ и время отправки содержимого в общий доступ к хранилищу параметров. Эти изменения автоматизируются в процессе обновления. Чтобы обновить агент UE-V Agent, запустите пакет установки агента UE-V (AgentSetup.exe, AgentSetupx86.msi или AgentSetupx64.msi) на компьютерах пользователей.

**Примечание.**  
При обновлении агента UE-V вы должны использовать один и тот же тип установщика (exe или MSI-пакет), который установил предыдущий агент UE-V. Например, чтобы обновить агенты UE-V 1,0, установленные с помощью AgentSetup.exe, используйте UE-V 2 AgentSetup.exe.



При запуске программы установки агента сохраняются следующие конфигурации:

-   Путь к хранилищу параметров

-   Параметры реестра

-   Запланированные задачи (параметры интервалов восстанавливаются по умолчанию)

**Примечание.**  
Компьютер с шаблонами расположений параметров UE-V 2. x, которые зарегистрированы в журнале событий Windows агента UE-V 1,0 Register Error.



Вы можете автоматизировать и распространить обновление агента UE-V с помощью Microsoft System Center 2012 Configuration Manager или другого средства распространения корпоративного программного обеспечения.

**Рекомендации:** Мы рекомендуем вам обновить все агенты UE-V 1,0 в вычислительной среде, но это не обязательно. Шаблоны расположений параметров UE-V 2. x могут взаимодействовать с агентом UE-V 1,0, так как они имеют доступ только к параметрам в пути к хранилищу параметров. Однако мы рекомендуем переносить развертывания на одну версию агента для упрощения управления и поддержки UE-V.

### Восстановление агента UE-V после неудачного обновления

При попытке выполнения одной из указанных ниже операций могут возникать ошибки.

-   Переход с UE-V 1,0 на UE-V 2

-   Переход на более новую версию Windows (например, с Windows 7 на Windows 8 или Windows 8 на Windows 8,1).

-   Удаление агента после обновления агента UE-V Agent

Для устранения проблем попробуйте восстановить агент UE-V, введя эту команду в командной строке на компьютере, на котором установлен агент.

``` syntax
msiexec.exe /f "<path to msi file>" /quiet /norestart /l*v "%temp%\UE-VAgentInstaller.log
```

Затем вы можете повторить процесс удаления или обновить, установив более новую версию агента UE-V Agent.
