# Управление административными резервными копиями и восстановлением в UE-V 2. x


Как администратор Microsoft User Experience Virtualization (UE-V) 2,0, 2,1 или 2,1 SP1, вы можете восстановить исходное состояние приложений и параметров Windows. Кроме того, в UE-V 2,1 можно также восстанавливать дополнительные параметры, когда пользователь применяет новое устройство.

## Параметры восстановления в UE-V 2,1 или UE-V 2,1 SP1, когда пользователь применяет новое устройство


Чтобы восстановить параметры, когда пользователь применяет новое устройство, вы можете добавить шаблон расположения параметров в профиль **резервного копирования** или **перемещения (по умолчанию)** с помощью командлета PowerShell Set-UevTemplateProfile. Это позволит синхронизировать параметры компьютера с новым компьютером в дополнение к параметрам пользователя. Шаблоны, назначенные профилю резервного копирования, заархивированы для этого устройства и настроены отдельно для каждого устройства. Чтобы выполнить резервное копирование параметров шаблона, используйте в Windows PowerShell следующий командлет:

``` syntax
Set-UevTemplateProfile -ID <TemplateID> -Profile <backup>
```

-   &lt;TemplateID &gt; является идентификатором шаблона UE-V

-   &lt;Резервное копирование &gt; может быть как резервным, так и перемещаемым

При замене устройства пользователя UE-V автоматически восстанавливает параметры, если домен пользователя, имя пользователя и имя устройства совпадают. Все синхронизированные данные и резервные копии будут восстановлены на устройстве автоматически.

Вы также можете использовать новый командлет PowerShell Restore-UevBackup для восстановления параметров с другого устройства. Чтобы клонировать пакеты параметров для нового устройства, используйте в Windows PowerShell следующий командлет:

``` syntax
Restore-UevBackup –Machine <MachineName>
```

где &lt; MachineName &gt; — это имя компьютера устройства.

Шаблоны, например шаблон Office 2013, включающий множество приложений, могут быть включены в перемещаемый профиль (по умолчанию) или в резервную копию. Отдельные приложения в наборе шаблонов следуют группе. Шаблоны из встроенных в Office 2013 включают как перемещаемые, так и параметры, заданные только для резервного копирования. В перемещаемый профиль нельзя включать параметры, заданные только для резервного копирования.

В рамках функции резервного копирования и восстановления UE-V добавил **последнюю известную подходящее (LKG)** к параметрам для возврата к параметрам. В этом выпуске вы можете вернуться к исходным параметрам или параметрам LKG. Параметры LKG позволяют пользователям возвращаться к промежуточной и стабильной точке после состояния предварительно UE-V параметров.

### Резервное копирование и восстановление шаблонов с помощью UE-V

Ниже указаны ключевые компоненты резервного копирования и восстановления UE-V:

-   Профили шаблонов

-   Расположение пакетов параметров в шаблоне места хранения параметров

-   Триггер резервного копирования

-   Восстановление параметров

**Профили шаблонов**

Профиль шаблона UE-V определяется при регистрации шаблона на устройстве или после регистрации с помощью служебной программы настройки PowerShell или WMI. Ниже перечислены типы профилей.

-   Роуминг (по умолчанию)

-   Архивация

-   На начальном выходе

Все шаблоны включаются в перемещаемый профиль при их регистрации, если не указано иное. Эти шаблоны синхронизируют параметры для всех устройств с поддержкой UE-V, для которых включен соответствующий шаблон.

Шаблоны можно добавить в профиль резервного копирования с помощью PowerShell или WMI, используя командлет Set-UevTemplateProfile. Шаблоны в резервной копии профиля. Создайте резервные копии этих параметров в хранилище параметров в специальном каталоге имени устройства. Указанные параметры заархивированы в указанном месте.

Шаблоны, помеченные как ненужные, включают в себя параметры, специфичные для этого устройства, которые не следует синхронизировать, пока не будет явно восстановлен. Эти параметры хранятся в том же расположении пакета параметров для определенного устройства в местоположении хранения параметров в качестве параметров backedup. Эти шаблоны содержат специальный идентификатор, внедренный в шаблон, который указывает, что они должны входить в этот профиль.

**Расположение пакетов параметров в шаблоне места хранения параметров**

Параметры перемещаемого профиля хранятся в местоположении хранения параметров. Шаблоны, назначенные для резервной копии или с помощью параметра "на один из профилей", хранят свои настройки в хранилище параметров в специальном каталоге имен устройств. Каждое устройство с шаблонами в этих профилях имеет собственное имя устройства. UE-V не очищает эти каталоги.

**Триггер резервного копирования**

Резервное копирование инициируется теми же событиями, которые инициируют синхронизацию UE-V.

**Восстановление параметров**

При восстановлении устройства пользователя восстанавливаются параметры зарегистрированного шаблона из папки резервного копирования другого устройства, а также все синхронизированные параметры с текущим компьютером. Параметры восстанавливаются следующими двумя способами:

-   **Автоматическое восстановление**

    Если пользовательский путь к хранилищу параметров UE-V, домен и имя компьютера совпадает с текущим пользователем, все параметры для этого пользователя синхронизируются с применением только последних параметров. Если пользователь в первый раз входит в систему на новом устройстве и эти условия выполняются, данные параметров применяются к этому устройству.

    **Примечание.**  
    Специальные возможности и параметры рабочего стола Windows требуют от пользователя повторного входа в Windows для применения.



-   **Восстановление вручную**

    Если вы хотите помочь пользователям с помощью восстановления устройства во время обновления, вы можете выбрать командлет Restore-UevBackup. Эта команда приводит к тому, что параметры пользователя загружаются из места хранения параметров.

## Восстановление параметров приложения и Windows в исходном состоянии


Команды WMI и Windows PowerShell позволяют восстанавливать параметры приложения и Windows в значениях параметров, которые были установлены на компьютере при первом запуске приложения после установки UE-V Agent. Это действие по восстановлению выполняется отдельно для каждого приложения или параметров Windows. Эти параметры восстанавливаются при следующем запуске приложения или восстанавливаются при входе пользователя в операционную систему.

**Восстановление параметров приложения и параметров Windows с помощью Windows PowerShell для UE-V 2. x**

1.  Откройте окно Windows PowerShell.

2.  Чтобы восстановить параметры приложения и параметры Windows, введите следующий командлет Windows PowerShell.
