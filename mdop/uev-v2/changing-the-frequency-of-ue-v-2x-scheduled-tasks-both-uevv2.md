# Изменение частоты выполнения запланированных задач UE-V 2. x


Установщик агента Microsoft User Experience Virtualization (UE-V) 2,0, 2,1 или 2,1 SP1 AgentSetup.exe создает следующие запланированные задачи при установке агента UE-V:

-   **Мониторинг параметров приложения**

-   **Приложение контроллера синхронизации**

-   **Синхронизация параметров при выходе из системы**

-   **Автоматическое обновление шаблона**

-   **Сбор данных по улучшению качества программного обеспечения**

-   **Отправка данных программы улучшения качества программного обеспечения**

**Примечание**  За исключением сбора данных CEIP, эти задачи должны оставаться включенными, так как UE-V не сможет работать без них.

 

Эти запланированные задачи не настраиваются с помощью средств UE-V. Администраторы, которые хотят изменить запланированную задачу для этих элементов, могут создать сценарий, который использует параметры командной строки Schtasks.exe.

Дополнительные сведения об Schtasks.exeх приведены в разделе [Использование программы Schtasks, exe для планирования задач в Windows Server 2003](https://go.microsoft.com/fwlink/?LinkID=264854).

## Запланированные задачи UE-V


Следующие запланированные задачи включены в UE-V 2 с помощью команд конфигурации планировщика заданий.

### Сбор данных по улучшению качества программного обеспечения

Если после установки пользователь или администратор выбрал участие в программе улучшения качества программного обеспечения (CEIP), UE-V собирает данные, чтобы улучшить продукт в будущих выпусках. Это запланированное задание запускается только при входе в систему. Задача **сбора данных CEIP** запускает UevSqmSession.exe, которая находится в каталоге установки агента UE-V.

### Мониторинг параметров приложения

Задача " **Параметры приложения** " используется для синхронизации параметров приложений для Windows. Оно запускается при входе в систему, но задерживается на 30 секунд, чтобы не повлиять на время входа в систему. Задача "мониторинг состояния приложения" запускает UevAppMonitor.exe файл, который находится в каталоге установки агента UE-V.

### Приложение контроллера синхронизации

Задача " **Синхронизация приложения контроллера** " используется для запуска контроллера синхронизации для синхронизации параметров с компьютера и места хранения параметров. По умолчанию задача выполняется каждые 30 минут. В это время локальные параметры синхронизируются с местом хранения параметров, и обновленные параметры в местоположении хранения параметров синхронизируются с компьютером. Приложение контроллера синхронизации запускает Microsoft.Uev.SyncController.exe, которое находится в каталоге установки UE-V Agent.
**Примечание.** Как и в случае с **параметрами приложения монитора** , эта задача запускается при входе в систему, но задерживается на 30 секунд, чтобы не повлиять на время входа в систему.


### Синхронизация параметров при выходе из системы

При выполнении задачи "Синхронизация параметров при выходе" запускается файл Microsoft.Uev.SyncController.exe, который находится в каталоге установки UE-V Agent.

### Автоматическое обновление шаблона

Задача " **Автоматическое обновление шаблона** " проверяет каталог шаблонов параметров на предмет новых, обновленных или удаленных шаблонов. Эта задача выполняется только в том случае, если настроена SettingsTemplateCatalog. Задача " **Автоматическое обновление шаблона** " запускает ApplySettingsCatalog.exe файл, который находится в каталоге установки агента UE-V.

### Отправка данных программы улучшения качества программного обеспечения

Задача « **Отправка данных CEIP** » запускается во время установки, если пользователь или администратор выбрал участие в программе улучшения качества программного обеспечения (CEIP). Эта задача передает данные на серверы программы улучшения качества программного обеспечения, в которых используются данные, чтобы улучшить продукт для будущих выпусков UE-V. Это запланированное задание запускается при входе в систему и каждые 4 часа позже. Задача « **Отправка данных CEIP** » запускает файл UevSqmUploader.exe, который находится в каталоге установки агента UE-V.

**Списка**

-   **Power Toggle** — планировщик заданий оптимизирует энергопотребление, если не подключен к сети переменного тока. Задача может перестать работать, если компьютер переключается на питание от батареи.

-   **Только в режиме бездействия** – задача будет остановлена, если компьютер прекратит работу. По умолчанию задача не будет перезапущена после простоя компьютера. Вместо этого задача начнется снова при следующем триггере задачи.

-   **Сетевое подключение** : задачи, помеченные как "Да", выполняются только в том случае, если на компьютере установлено сетевое подключение. Задачи, помеченные как "н/д", работают независимо от сетевого подключения.

### Управление запланированными задачами

Чтобы найти запланированные задачи, выполните указанные ниже действия.

1.  Откройте "Расписание задач" на компьютере пользователя.

2.  Перейти к: планировщик заданий — &gt; Библиотека планировщика заданий — &gt; Microsoft- &gt; UE-V

3.  Выберите запланированную задачу, которую вы хотите настроить, и настройте ее в области сведений.

### Дополнительные сведения

Следующие дополнительные сведения применимы к запланированным задачам UE-V.

-   Все программы для последовательности задач находятся в папке установки агента UE-V (по `%programFiles%\Microsoft User Experience Virtualization\Agent\[architecture]\` умолчанию).

-   Запланированная задача приложения контроллера синхронизации является важнейшим компонентом, если UE-V PowerSchool имеет значение "SyncProvider" (конфигурация по умолчанию UE-V 2). Это запланированное задание синхронизирует SettingsSToragePath с локально кэшированными версиями файлов пакетов параметров. Если пользователи приводят к нечастой синхронизации параметров, вы можете уменьшить количество запланированных задач так, чтобы оно было меньше 1 минуты.При необходимости вы также можете увеличить значение по умолчанию: 30 мин на более высокое значение. Если пользователи выдерживают, что параметры не синхронизируются достаточно быстро при входе в систему, вы можете удалить параметр задержки для запланированной задачи. (Параметры задержки можно найти в диалоговом окне " **изменение триггера** ")

-   Вы не хотите отключать запланированную задачу "автоматическое обновление шаблона", если вы используете другой метод для синхронизации шаблонов клиентов (например, групповой политики или базовых планов Configuration Manager). Если значение свойства SettingsTemplateCatalog оставлено пустым, UE-V не проверяет каталог параметров для настраиваемых шаблонов. Это запланированное задание запускается ApplySettingsCatalog.exe и, в сущности, будет возвращаться немедленно.

-   Запланированная задача "Параметры приложения" обновляет параметры приложения для Windows (AppX) в режиме реального времени, основываясь на параметрах программы приложения Windows, встроенных в каждое приложение.
