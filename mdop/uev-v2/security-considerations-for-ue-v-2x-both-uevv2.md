# Рекомендации по безопасности для UE-V 2.x


В этой статье содержится краткий обзор учетных записей и групп, файлов журналов и других вопросов, связанных с безопасностью, для Microsoft User Experience Virtualization (UE-V) 2,0, 2,1 и 2,1 SP1. Чтобы получить дополнительные сведения, воспользуйтесь приведенными ниже ссылками.

## Рекомендации по безопасности для конфигурации UE-V


**Важно!**  При создании общего доступа к хранилищу параметров Ограничьте общий доступ для пользователей, которым требуется доступ.

 

Так как пакеты параметров могут содержать персональные данные, необходимо соблюдать осторожность и защитить их. Как правило, выполните указанные ниже действия.

-   Ограничьте общий доступ только теми пользователями, которым требуется доступ. Создайте группу безопасности для пользователей, которые перенаправляли папки в определенном общем доступе и ограничивают доступ только тем пользователям.

-   После создания общего доступа Скройте общий доступ, указав $ после имени общего доступа. Это добавление скрывает общий доступ из браузеров, и этот общий доступ не отображается в окне Мое сетевое окружение.

-   Предоставление пользователям минимального количества разрешений, которые они должны иметь. В приведенных ниже таблицах показаны необходимые разрешения.

    1.  Установите следующие разрешения SMB уровня общего доступа для папки "Настройка места хранения".

        | Учетная запись пользователя | Рекомендуемые разрешения |
        | - | - |
        | Все пользователи | Нет разрешений |
        |Группа безопасности UE-V | Полный доступ |

    2.  Установите следующие разрешения файловой системы NTFS для папки "место хранения параметров".

        | Учетная запись пользователя | Рекомендуемые разрешения | Папка |
        | - | - | - |
        | Создатель/владелец | Полный доступ | Только для вложенных папок и файлов|
        | Администраторы домена | Полный доступ | Эта папка, вложенные папки и файлы |
        | Группа безопасности пользователей UE-V | Просмотр папки/Чтение данных, создание папок и добавление данных | Только для этой папки |
        | Все пользователи | Удаление всех разрешений | Нет разрешений |

    3.  Установите следующие разрешения SMB уровня общего доступа для папки каталога шаблонов параметров.

        | Учетная запись пользователя | Рекомендации по разрешениям |
        | - | - |

        | Компьютеры домена | Уровни разрешений "чтение" |
        | Администраторы | Уровни разрешений на чтение и запись |
         
    4.  Установите следующие разрешения NTFS для папки каталога шаблонов параметров.

        | Учетная запись пользователя | Рекомендуемые разрешения | Применить к |
        | - | - | - |
        | Создатель/владелец | Полный доступ | Эта папка, вложенные папки и файлы |
        | Компьютеры домена | Список содержимого папки и разрешения на чтение | Эта папка, вложенные папки и файлы|

### Использование WindowsServer AS из WindowsServer2003 для перенаправленных файловых ресурсов hosts

Файлы пакета параметров пользователя содержат персональные данные, которые передаются между клиентским компьютером и сервером, на котором хранятся пакеты параметров. Из-за этого процесса вы должны убедиться, что данные защищены при пересылке по сети.

Данные о параметрах пользователей уязвимы для этих потенциальных угроз: перехват данных по мере их появления в сети, искажение данных по мере их появления в сети, а также подмена сервера, на котором размещены данные.

Как и в Windows server2003, некоторые возможности операционной системы Windows Server могут помочь защитить пользовательские данные.

-   **Kerberos** – Standard для всех версий Microsoft Windows2000Server и WindowsServer, начиная с WindowsServer2003. Kerberos обеспечивает наивысший уровень безопасности сетевых ресурсов. NTLM проверяет подлинность клиента; Kerberos проверяет подлинность сервера и клиента. При использовании NTLM клиент не знает, является ли сервер допустимым. Эта разница особенно важна, если клиент обменивается личными файлами с сервером, как в случае с перемещаемыми профилями пользователей. Протокол Kerberos обеспечивает более высокую безопасность, чем NTLM. Протокол Kerberos недоступен в операционных системах Microsoft WindowsNTServer 4,0 или более ранних версий.

-   **IPSec** — протокол IP-безопасности (IPSec) обеспечивает проверку подлинности на уровне сети, целостность данных и шифрование. IPsec обеспечивает следующее:

    -   Перемещаемые данные изменяются в целях защиты данных, когда данные направляются по короткому маршруту.

    -   Перемещенные данные можно безопасно отсекать, просматривать и копировать.

    -   Перемещаемые данные безопасно получать из Access неавторизованными сторонами.

-   **Подписывание SMB** — протокол проверки подлинности протокола SMB поддерживает проверку подлинности сообщений, что предотвращает использование активного сообщения и атаки "злоумышленник в середине". Подписывание SMB обеспечивает такую проверку подлинности, помещая цифровую подпись в каждый из протоколов SMB. После этого цифровая подпись будет проверяться клиентом и сервером. Чтобы использовать подписывание SMB, необходимо сначала включить его либо только на клиенте SMB, но и на сервере SMB. Обратите внимание, что при подписывании SMB накладывается снижение производительности. Она не потребляет больше пропускной способности сети, но использует больше тактов ЦП на стороне клиента и сервера.

### Всегда используйте файловую систему NTFS для томов, содержащих данные пользователей.

Для наиболее безопасной конфигурации настройте серверы, на которых размещены файлы параметров UE-V, для использования файловой системы NTFS. В отличие от файловой системы FAT, NTFS поддерживает избирательные списки управления доступом (DACL) и списки управления доступом системы (SACL). Списки DACL и SACL управляют тем, кто может выполнять операции с файлом, а также события, запускающющие ведение журнала действий, выполняемых над файлом.

### Не полагаться на шифрование EFS для шифрования файлов пользователей при их передаче по сети

При использовании шифрованной файловой системы (EFS) для шифрования файлов на удаленном сервере зашифрованные данные не шифруются при передаче по сети; оно шифруется только в том случае, если оно хранится на диске.

Этот процесс шифрования не применяется, если система включает IP-безопасность (IPsec) или веб-сайт и службу управления версиями (WebDAV). IPsec шифрует данные при транспортировке в сети TCP/IP. Если файл зашифрован перед копированием или перемещением в папку WebDAV на сервере, он остается зашифрованным во время передачи и хранится на сервере.

### Разрешите агенту UE-V создавать папки для каждого пользователя

Чтобы обеспечить оптимальное функционирование UE-V, создайте только корневой общий доступ на сервере и предоставьте агенту UE-V создание папок для каждого пользователя. UE-V создает эти папки пользователей с соответствующей защитой.

Эта конфигурация разрешений позволяет пользователям создавать папки для хранения параметров. Агент UE-V создает и защищает папку пакета параметров при ее запуске в контексте пользователя. Пользователи получают полный доступ к папке пакетов параметров. Другие пользователи не наследуют доступ к этой папке. Вам не нужно создавать и защищать отдельные каталоги пользователей. Агент, который запускается в контексте пользователя, делает это автоматически.

**Примечание**  Дополнительную защиту можно настроить при использовании Windows Server для общего доступа к хранилищу параметров. UE-V можно настроить, чтобы убедиться, что либо локальная группа администраторов, либо текущий пользователь является владельцем папки, в которой хранятся пакеты параметров. Чтобы включить дополнительную защиту, введите следующую команду:

1.  Добавьте раздел реестра REG \ _DWORD RepositoryOwnerCheckEnabled `HKEY_LOCAL_MACHINE\Software\Microsoft\UEV\Agent\Configuration` .

2.  Установите для раздела реестра значение *1*.

Если этот параметр конфигурации установлен, агент UE-V Agent удостоверяет, что локальная группа администраторов или текущий пользователь является владельцем папки пакета параметров. В противном случае агент UE-V не предоставляет доступ к этой папке.

 

Если необходимо создать папки для пользователей, убедитесь в том, что у вас установлены правильные разрешения.

Настоятельно рекомендуется не создавать папки предварительно. Вместо этого позвольте агенту UE-V создать папку для пользователя.

### Предоставление разрешений на хранение параметров UE-V 2 в основном каталоге или в настраиваемой папке

Если вы перенаправляли параметры UE-V в основной каталог пользователя или в пользовательский каталог Active Directory, убедитесь в том, что разрешения для каталога установлены надлежащим образом для вашей организации.
