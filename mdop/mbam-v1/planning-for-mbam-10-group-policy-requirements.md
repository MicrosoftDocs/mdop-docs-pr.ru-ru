# Планирование требований групповой политики MBAM 1.0


Для управления клиентами администрирования и наблюдения (MBAM) Microsoft BitLocker требуются пользовательские параметры групповой политики, которые необходимо применить. В этом разделе описаны доступные параметры политики для объекта групповой политики (GPO), которые можно использовать для управления шифрованием диска BitLocker в предприятии с помощью MBAM.

**Важно.**  
MBAM не использует параметры GPO по умолчанию для шифрования диска Windows BitLocker. Если включены параметры по умолчанию, это может привести к конфликтам поведения. Чтобы включить MBAM для управления BitLocker, необходимо задать параметры политики GPO после установки шаблона групповой политики MBAM.



После установки шаблона групповой политики MBAM вы можете просматривать и изменять доступные пользовательские параметры политики MBAM, которые позволят MBAM управлять шифрованием Enterprise BitLocker. Шаблон групповой политики MBAM должен быть установлен на компьютере, на котором может выполняться консоль управления групповыми политиками (GPMC) или Расширенная технология MDOP для управления групповыми политиками. Затем, чтобы изменить применимый объект групповой политики, откройте консоль управления групповыми политиками (GPMC) и перейдите к следующему узлу GPO: **конфигурации компьютера**, \\ **шаблоны**, \\ **компоненты Windows** \\ **MDOP MBAM (Управление BitLocker)**.

Узел GPO MDOP MBAM (BitLocker Management) включает четыре параметра глобальных политик и четыре дочерних узла параметров GPO соответственно. В параметрах глобальных политик GPO используются следующие параметры: Управление клиентами, несъемные диски, диски операционной системы и съемные диски. В следующих разделах приведены определения политик и рекомендуемые параметры политики, которые помогут вам спланировать требования к параметрам политики MBAM GPO.

**Примечание.**  
Дополнительные сведения о настройке минимально предложенных параметров GPO для включения MBAM управления шифрованием BitLocker вы узнаете, [как изменить параметры GPO MBAM 1,0](how-to-edit-mbam-10-gpo-settings.md).

<p>Настройте эту политику, чтобы сделать следующее:</p>

<li><p>Принудительная защита BitLocker для диска с операционной системой.</p></li>
<li><p>Настройте использование закрепления с помощью ПИН-кода доверенного платформенного модуля (TPM) для защиты операционной системы.</p></li>
<li><p>Настройка расширенных загрузочных закрепления для разрешения знаков, таких как прописные и строчные буквы и цифры. MBAM не поддерживает использование символов и пробелов для расширенных контактов, несмотря на то что BitLocker поддерживает символы и пробелы.</p></li>
</ul>
<p>Если вы включите этот параметр политики, пользователи обязаны защищать диск операционной системы с помощью BitLocker.</p>
<p>Если вы не настраиваете и не можете отключить этот параметр, пользователи не обязаны защищать диск операционной системы с помощью BitLocker.</p>
<p>Если этот параметр отключен, агент MBAM расшифровывает том операционной системы, если он зашифрован.</p>
<p>Если этот параметр политики включен, пользователям необходимо обеспечить защиту операционной системы с помощью BitLocker, а диск зашифрован. В соответствии с вашими требованиями к шифрованию вы можете выбрать способ защиты для диска с операционной системой.</p>
<p>Для более высоких требований к безопасности используйте доверенный платформенный модуль + PIN-код, разрешите Расширенные ПИН-коды и установите для них минимальную длину в восьми символов.</p>
<p>Если эта политика включена с предохранителем доверенного платформенного модуля +, вы можете отключить следующие политики в <strong> разделе </strong>  /  <strong> </strong>  /  <strong> Параметры режима сна системного управления питанием </strong> :</p>
<ul>
<li><p>Разрешить режимы ожидания (S1-S3) при простое компьютера (питание от сети)</p></li>
<li><p>Разрешить режимы ожидания (S1-S3) при простое компьютера (питание от батареи)</p></li>
</ul></td>

Этот параметр политики не применяется, если на компьютере не установлен совместимый доверенный платформенный модуль или для BitLocker уже включена защита TPM.</p>
<p>Если эта политика не настроена, доверенный платформенный модуль использует профиль проверки платформы по умолчанию или профиль проверки платформы, указанный в сценарии настройки.</p></td>

<td align="left"><p>Предлагаемая конфигурация: <strong> не настроена</strong></p>
