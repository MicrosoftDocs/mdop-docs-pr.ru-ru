# Рекомендации по безопасности для MBAM 1.0


В этой статье содержится краткий обзор учетных записей и групп, файлов журналов и других вопросов, связанных с безопасностью, для администрирования и мониторинга Microsoft BitLocker (MBAM). Чтобы получить дополнительные сведения, воспользуйтесь ссылками, приведенными в этой статье.

## Общие рекомендации по безопасности


**Общие сведения о рисках для системы безопасности.** Наиболее серьезный риск для MBAM заключается в том, что его функциональность может захватить неавторизованный пользователь, который может затем заново настроить шифрование BitLocker и получить данные ключа шифрования BitLocker на клиентах MBAM. Однако утрата функций MBAM в течение короткого промежутка времени из-за атаки типа "отказ в обслуживании" обычно не вызывает катастрофического воздействия.

**Физическая защита компьютеров**. Безопасность не завершена без физического обеспечения безопасности. Любой пользователь, имеющий физический доступ к серверу MBAM, может вызвать атаку на всю клиентскую базу. Любые потенциальные физические атаки следует учитывать в случае высокой опасности и устранения соответствующих проблем. Серверы MBAM должны храниться в физически надежной серверной комнате с управляемым доступом. Защитите эти компьютеры, если администраторы не находятся в физическом режиме, с помощью операционной системы и запирания компьютера или защищенной экранной заставки.

**Установка последних обновлений для системы безопасности на всех компьютерах**. Получайте сведения о новых обновлениях для операционных систем, Microsoft SQL Server и MBAM, подписавшись на службу уведомлений системы безопасности ( <https://go.microsoft.com/fwlink/p/?LinkId=28819> ).

**Используйте надежные пароли и передавайте фразы**. Для всех учетных записей администраторов MBAM и MBAM следует использовать надежные пароли с 15 и более знаками. Никогда не используйте пустые пароли. Дополнительные сведения о концепциях паролей можно найти в статье "пароли и политики учетных записей" на сайте TechNet ( <https://go.microsoft.com/fwlink/p/?LinkId=30009> ).

### Учетная запись для доступа к отчетам SSRS

Учетная запись службы отчетов служб SQL Server Reporting Services (SSRS) предоставляет контекст безопасности для выполнения отчетов MBAM, доступных через службы SSRS. Эта учетная запись настраивается при настройке MBAM.

## Файлы журнала MBAM


В процессе настройки MBAM в папке% TEMP% для пользователя, устанавливающего эту версию, создаются следующие файлы журнала установки.

**Файлы журнала установки MBAM Server**

<a href="" id="msi-five-random-characters--log"></a>MSI <em> &lt; пять случайных знаков &gt; </em> . log  
Записываются в журнал действия, выполненные во время установки MBAM и установки компонентов сервера MBAM.

<a href="" id="installcompliancedatabase-log"></a>InstallComplianceDatabase. log  
Заносит в журнал действия, предпринятые для создания настройки базы данных "состояние соответствия MBAM".

<a href="" id="installkeycompliancedatabase-log"></a>InstallKeyComplianceDatabase. log  
Заносит в журнал действия, предпринятые для создания базы данных восстановления MBAM и конфигурации оборудования.

<a href="" id="addhelpdeskdbauditusers-log"></a>AddHelpDeskDbAuditUsers. log  
Заносит в журнал действия, предпринятые для создания имен входа SQL Server в базе данных состояния соответствия MBAM и авторизации веб-службы поддержки в базу данных для отчетов.

<a href="" id="addhelpdeskdbusers-log"></a>AddHelpDeskDbUsers. log  
Заносит в журнал действия, предпринятые для авторизации веб-служб в базу данных для восстановления ключей и создают имена для входа в базу данных восстановления MBAM и оборудования.

<a href="" id="addkeycompliancedbusers-log"></a>AddKeyComplianceDbUsers. log  
Заносит в журнал действия, предпринятые для авторизации веб-служб на MBAM базы данных состояния соответствия требованиям для создания отчетов о соответствии.

<a href="" id="addrecoveryandhardwaredbusers-log"></a>AddRecoveryAndHardwareDbUsers. log

## Вопросы TDE базы данных MBAM


Функция шифрования прозрачных данных (TDE), доступная в SQL Server 2008, является обязательным условием установки для экземпляров базы данных, которые будут размещаться в MBAMных функциях базы данных.

С помощью TDE можно выполнять полное шифрование на уровне базы данных в режиме реального времени. TDE является наилучшим вариантом массового шифрования для соответствия требованиям законодательства и корпоративным стандартам безопасности данных. TDE работает на уровне файла, который аналогичен двум функциям Windows: шифрованной файловой системе (EFS) и шифрованию диска BitLocker, в которых также шифруются данные на жестком диске. TDE не заменяет шифрование на уровне ячейки, EFS и BitLocker.

Если в базе данных включена TDE, все резервные копии шифруются. Таким образом, чтобы убедиться в том, что сертификат, использовавшийся для защиты ключа шифрования базы данных (DEK), должен быть архивирован и поддерживаться в резервной копии базы данных. Без сертификата данные будут нечитаемы. Создавайте резервные копии сертификата вместе с базой данных. Каждая резервная копия сертификата должна иметь два файла; Оба этих файла должны быть архивированы. Рекомендуется архивировать их отдельно от файла резервной копии базы данных для обеспечения безопасности.

Пример включения TDE для экземпляров базы данных MBAM можно найти в разделе [Оценка MBAM 1,0](evaluating-mbam-10.md).

Дополнительные сведения о TDE в SQLServer 2008 можно найти [в разделе Шифрование базы данных в SQL server 2008 Enterprise Edition](https://go.microsoft.com/fwlink/?LinkId=269703).
