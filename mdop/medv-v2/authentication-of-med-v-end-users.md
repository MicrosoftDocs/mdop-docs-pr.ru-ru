# Аутентификация конечных пользователей MED-V


Проверка подлинности конечных пользователей Microsoft Enterprise Virtualization (MED-V) 2,0 для конечного пользователя является важной проблемой системы безопасности. В этом контексте проверка подлинности — проверка удостоверения конечного пользователя MED-V.

В следующем разделе представлена информация и руководство по проверке подлинности конечными пользователями в MED-V.

## Проверка подлинности пользователей в MED-V


Проверка подлинности в MED-V обычно выполняется на двух уровнях: когда пользователь впервые получает доступ к MED-V и каждый раз при изменении пароля.

В зависимости от того, как вы настроили параметры для проверки подлинности в службах MED-V, конечные пользователи обычно запрашиваются в какой-либо момент для ввода пароля либо при первом запуске MED-V или при первом попытке открыть опубликованное приложение.

Есть несколько аспектов проверки подлинности конечных пользователей, которые можно контролировать, включая указанные ниже.

Хранятся ли в диспетчере учетных данных учетные данные, которые пользователь вводит.

В каком способе отображается конечный пользователь с возможностью ввода и сохранения пароля

В зависимости от предпочтительного процесса для управления проверкой подлинности конечными пользователями вы можете указать, следует ли выполнять кэширование учетных данных для конкретной рабочей области MED-V. Кэширование учетных данных конечного пользователя полезно, так как они запрашиваются только один раз для ввода пароля. Если пользователю не разрешено сохранять пароль или это не решит, что каждый раз при запуске нового сеанса MED-V нужно будет ввести его еще раз. Например, если MED-V настроен на запуск при входе пользователя в систему, но проверка подлинности отключена, конечные пользователи будут запрашиваться только один раз во время входа. В этом случае учетные данные действительны до тех пор, пока пользователь не завершит сеанс с компьютера.

При необходимости вы можете использовать диспетчер учетных данных для удаления всех сохраненных учетных данных для конечных пользователей.

По умолчанию хранение учетных данных отключено, но вы можете изменить этот параметр, выполнив одно из указанных ниже действий.

**При создании пакета рабочей области для MED-V**. Дополнительные сведения можно найти [в разделе Создание пакета рабочей области для MED-V](create-a-med-v-workspace-package.md).

**После развертывания рабочей области MED-V**. Измените параметр командлета MED-V UxCredentialCacheEnabled, чтобы установить раздел реестра служб терминалов. Дополнительные сведения можно найти в справке Windows PowerShell.

После развертывания рабочей области MED-V вы можете настроить параметры проверки подлинности для конечных пользователей, изменив политику служб терминалов с именем DisablePasswordSaving. DisablePasswordSaving определяет, отображается ли флажок Сохранить пароль в диалоговом окне клиента RDP и отображается ли запрос учетных данных MED-V.

Ниже указан путь политики для политики служб терминалов с именем DisablePasswordSaving.

**Программа**

HKEY \ _LOCAL \ _MACHINE \\SOFTWARE\\Microsoft\\Virtual Machine\\Policies\\DisablePasswordSaving

**Примечание.**  
Изменения, вносимые в DisablePasswordSaving, влияют только на запрос RDP для виртуальной машины.

По умолчанию установка MED-V устанавливает раздел реестра гостя, чтобы отключить приглашение "срок действия пароля истек". Пользователю будет предложено только сменить пароль на узле. Учетные данные, обновленные на узле, передаются на гость.

**Осторожны**  
Если в вашей среде используется групповая политика, необходимо знать, что она может переопределять раздел реестра, что приводит к появлении запроса на ввод пароля от гостя.



### Вопросы безопасности при проверке подлинности

Несмотря на то, что кэширование учетных данных конечного пользователя обеспечивает наилучшее взаимодействие с пользователем, необходимо учитывать риски, связанные с этим.

Если включено кэширование учетных данных, учетные данные домена конечного пользователя хранятся в диспетчере учетных данных Windows в виде обратимого формата. В результате злоумышленник может написать средство, работающее как процесс системного уровня или процесс конечного пользователя, и получить учетные данные конечного пользователя. Вы можете уменьшить этот риск, установив для DisablePasswordSaving значение **Enabled**.

Эта проблема возникает, когда проверка подлинности MED-V отключена, но параметр политики служб терминалов включен.
