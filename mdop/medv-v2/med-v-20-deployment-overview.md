# Обзор развертывания MED-V 2.0


В этом разделе приводятся общие сведения и инструкции по установке и развертыванию Microsoft Enterprise Virtualization (MED-V) 2,0.

## Обзор


MED-V 2,0 основывается на модели приложения, в которой те же методы, которые используются для развертывания приложений, можно использовать для развертывания и управления MED-V. Развернутое решение для MED-V включает два компонента: агент узла MED-V и гостевой агент. Агент узла MED-V установлен на рабочем столе Windows 7, а Гостевой агент MED-V установлен в Windows XP в рабочей области MED-V. MED-V также включает в себя диспетчер рабочих областей MED-V, который предоставляет сведения и инструменты, необходимые для создания и настройки рабочих областей MED-V.

**Важно.**  
В целях MED-V для всех пользователей поддерживается только установка диспетчера рабочих областей MED-V, агента хоста MED-v и рабочей области MED-V. Установка MED-V для текущего пользователя только путем выбора параметра **ALLUSERS = ""** приводит к сбоям при установке компонентов и настройке рабочей области MED-V.



### Файлы установки для MED-V

MED-V включает следующие установочные файлы, необходимые для работы с MED-V.

**Установочный файл агента узла MED-V**

Установочный файл агента узла имеет имя MED-V\_HostAgent\_Setup.exe. Этот файл распространяется на все подходящие компьютеры конечных пользователей в рамках развертывания MED-V на уровне предприятия.

**Установочный файл упаковщика рабочей области для MED-V**

Установочный файл упаковщика рабочей области MED-V имеет имя MED-V\_WorkspacePackager\_Setup.exe. Используйте этот файл, чтобы установить диспетчер рабочих областей MED-V на компьютер, на котором у вас есть права и разрешения администратора. Администратор на рабочем столе использует диспетчер рабочих областей MED-V для создания рабочих областей MED-V и управления ими.

**Примечание.**  
Гостевой агент MED-V устанавливается автоматически при первом запуске программы установки.



### Процесс развертывания MED-V

Ниже приведен общий обзор процесса установки и развертывания для MED-V.

1.  Установите диспетчер рабочих областей MED-V на компьютере, на котором у вас есть учетные данные администратора, и что вы будете использовать для создания пакетов рабочей области для MED-V. Дополнительные сведения можно найти [в разделе Установка диспетчера рабочих областей MED-V](how-to-install-the-med-v-workspace-packager.md).

2.  Подготовьте образ MED-V и создайте пакеты рабочей области для MED-v с помощью диспетчера рабочих областей MED-V. Дополнительные сведения можно найти в разделе [операции для MED-V](operations-for-med-v.md).

3.  Развертывайте необходимые компоненты MED-V во всей Организации. В качестве обязательных компонентов MED-V используются виртуальные компьютеры Windows, агент хоста MED-V и Рабочая область MED-V.

**Важно.**  
Установка компонентов MED-V требует учетных данных администратора. Если конечный пользователь устанавливает MED-V, ему будет предложено ввести учетные данные администратора. Кроме того, учетные данные администратора могут быть предоставлены в контексте, если установка выполняется с помощью электронной системы программного обеспечения (ESD).



### Компоненты MED-V

Компоненты MED-V, развертываемые в рамках всего предприятия, состоят из следующих компонентов:

**Windows Virtual PC**

Функции MED-V в изображениях виртуальных ПК на основе Windows для решения совместимости. Windows Virtual PC и обновление для Windows 7 (KB977206) являются обязательными. Дополнительные сведения можно найти в разделе [Настройка предварительных требований для установки](configure-installation-prerequisites.md).

**Установочный файл агента узла MED-V**

MED-V\_HostAgent\_Setup.exe.

**Установочные файлы рабочей области для MED-V**

Установочные файлы рабочей области MED-V создаются при создании пакета рабочей области MED-V, который включает следующие возможности:

Исполняемая программа setup.exe, выполняющая установку рабочей области MED-V

&lt;Установщик MED-V \ _workspace \ _name &gt; . msi

&lt;Файл VHD: _filename &gt; . MEDV, который является сжатым виртуальным жестким диском

Файлы параметров конфигурации ( &lt; Рабочая область \ _name &gt; . reg и &lt; Workspace \ _name &gt; . ps1)

Чтобы развернуть MED-V, скопируйте все необходимые файлы установки на узел компьютера или в общий доступ, доступ к которому может получить узел компьютера. Запустите файлы установки компонента для Windows Virtual PC, агента хостов MED-V и рабочей области MED-V. Затем запустите агент узла MED-V, чтобы выполнить первый раз при установке MED-V.

Вы можете выполнить установку вручную. Однако мы рекомендуем использовать электронный метод распространения программного обеспечения для автоматизации развертывания компонентов. Дополнительные сведения можно найти в разделе [Развертывание рабочей области MED-V с помощью электронной системы распространения программного обеспечения](how-to-deploy-a-med-v-workspace-through-an-electronic-software-distribution-system.md).

**Примечание.**  
Сведения о доступных аргументах командной строки для управления параметрами установки можно найти в разделе [Параметры командной строки для установочных файлов MED-V](command-line-options-for-med-v-installation-files.md).



## Действия по развертыванию


При развертывании MED-V по всему предприятию необходимо учитывать два основных аспекта: Установка и первая Настройка.

### Установка

1.  **Windows Virtual PC** – во время установки для Windows 7 (KB977206) на компьютере с операционной системой MED-V проверяются виртуальные компьютеры Windows и необходимые обновления. Дополнительные сведения можно найти в разделе [Настройка предварительных требований для установки](configure-installation-prerequisites.md).

    Вы можете установить эти компоненты в установках Windows 7 перед установкой MED-V или установить их в рамках дистрибутива MED-V. Однако MED-V не включает механизм для развертывания; они должны быть развернуты с помощью электронного распространения программного обеспечения или в составе образа Windows 7.

    **Важно.**  
    При установке компонентов MED-V с помощью пакетного файла рекомендуется указать, что Windows Virtual PC и Windows Virtual PC Hotfix устанавливаются после агента узла MED-V и файлов пакета MED-V Workspace. Это означает, что при установке центра обновления Windows не будет вмешательство в процесс установки, если требуется перезагрузка.



~~~
**Note**  
After you install Windows Virtual PC, the computer must be restarted.
~~~



2. **Агент узла MED-v** — Установка агента узла MED-v на компьютере с Windows 7, на котором будет выполняться med-v. Этот параметр должен быть установлен перед установкой рабочей области MED-V и проверяет, установлен ли Windows Virtual PC.

3. **Рабочая область MED-v** вы создаете файлы, необходимые в этой установке с помощью диспетчера рабочих областей med-v: setup.exe, MEDV и MSI-файлы. Чтобы установить рабочую область для MED-V, запустите setup.exe; При этом другие файлы перезапускаются должным образом. В процессе установки в реестре под ключом запуска на локальном компьютере заносится запись, с помощью которой запускается агент хоста MED-v, который всегда запускает MED-V при запуске Windows.

   **Важно.**  
   Установка рабочей области MED-V может выполняться конечным пользователем или автоматически с помощью электронной системы распространения программного обеспечения. Для установки рабочей области MED-V требуются учетные данные администратора, поэтому конечные пользователи должны быть администраторами компьютеров для установки рабочей области MED-V. Кроме того, электронная система распространения программного обеспечения обычно работает в контексте системы и обладает достаточными разрешениями.



~~~
**Tip**  
Because of problems that can occur when you install MED-V from a network location, we recommend that you copy the MED-V workspace setup files locally and then run setup.exe.
~~~



### Установка первого раза

После установки MED-V и обязательных компонентов необходимо настроить MED-V. Настройка MED-V известна при первом запуске программы установки. С помощью **диспетчера рабочих областей MED-V**вы можете настроить первый запуск программы установки в автоматическом режиме. При первой настройке MED-V требуется, чтобы конечные пользователи могли ввести пароль для проверки подлинности в рабочей области MED-V, но в противном случае она может быть практически невидимой для пользователя. Уведомления отображаются в области уведомлений, например при первом завершении настройки и готовности приложений. Ниже приведены действия, которые выполняются при первом запуске MED-V.

1.  Необходимо настроить виртуальный жесткий диск. Мини-установка запускается и разворачивает образ Windows XP. Обычно это происходит в скрытом окне, но MED-V можно настроить для отображения во время этой конфигурации.

2.  После завершения мини-установки вы можете выполнять команды, необходимые для дополнительной настройки, такие как установка программного обеспечения ESD или других приложений, или Настройка изображения. Они могут быть вызваны в файле Sysprep. INF, но это не обязательно. Дополнительные сведения можно найти [в разделе Настройка образа ВИРТУАЛЬНЫХ ПК Windows для MED-V](configuring-a-windows-virtual-pc-image-for-med-v.md).

3.  Ftscompletion.exe выполняется на последнем этапе настройки. Этот процесс завершает настройку MED-V, добавляет пользователя в группу RDP для доступа к рабочей области MED-V, копирует журналы, сигналы MED-V, которые готовы к работе с рабочей областью MED-v, а затем перезапускает рабочую область MED-V. Этот процесс также может добавить пользователя в качестве администратора рабочей области MED-V, если он был настроен при создании рабочей области MED-V. Ftscompletion.exe обычно вызывается с помощью Sysprep, INF-файла, но также может выполняться с помощью другого метода, например сценария. Однако Ftscompletion.exe должны быть последними действиями, которые выполняются при настройке рабочей станции. Дополнительные сведения можно найти [в разделе Настройка образа ВИРТУАЛЬНЫХ ПК Windows для MED-V](configuring-a-windows-virtual-pc-image-for-med-v.md).

4.  После того, как Рабочая область MED-V перезапустится Ftscompletion.exe, конечный пользователь войдет в систему. Если вы не сохранили свой пароль в первый раз, вам будет предложено ввести его еще раз. Затем Рабочая область MED-V запускается и конфигурируется для пользователя. Конфигурация включает применение групповой политики.

    Рекомендуется применять только те политики, которые имеют смысл в среде совместимости приложений для Windows XP. Например, политики персонализации рабочего стола обычно не нужно применять и должны быть отключены. Дополнительные сведения о том, как разрешить только локальные профили, можно найти в разделе [Параметры групповой политики для перемещаемых профилей пользователей](https://go.microsoft.com/fwlink/?LinkId=205072) ( https://go.microsoft.com/fwlink/?LinkId=205072) .

После того как вы закончите настройку, конечные пользователи получат уведомление о том, что опубликованные приложения готовы. После этого они смогут получать доступ к приложениям, установленным в рабочей области MED-V, из меню " **Пуск** ".
