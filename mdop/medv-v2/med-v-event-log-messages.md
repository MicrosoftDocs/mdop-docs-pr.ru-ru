# Сообщения журнала событий MED-V


Файлы журналов для Microsoft Enterprise Virtualization (MED-V) 2,0 содержат подробные сведения о том, как развертывать MED-V и управлять ими в вашей организации, а также помогают устранять проблемы.

## Коды событий


Ниже приведен список кодов событий MED-V, помогающих устранить проблемы, которые могут возникнуть при развертывании и управлении MED-V.

### Fts

Коды событий при первом запуске программы установки.

### Событие с кодом 3066

Не удалось запустить операцию виртуальной машины.

<a href="" id="description"></a>**Описание**  
Существует потенциальная проблема с виртуальным жестким диском (VHD), который вы используете для создания рабочей области MED-V.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что вы можете создать виртуальную машину с помощью виртуального жесткого диска для MED-V и что она может быть запущена.

### Событие с кодом 3071

Подготовка виртуальной машины завершилась сбоем.

<a href="" id="description"></a>**Описание**  
Произошла ошибка при первом запуске программы установки, которая может быть вызвана многими разными проблемами. Сюда входят проблемы с подключением к сети.

<a href="" id="solution"></a>**Решение**  
Перезапустите агент узла MED-V, чтобы запустить программу установки в первый раз.

### Событие с кодом 3078

Подготовка виртуальной машины завершилась сбоем.

<a href="" id="description"></a>**Описание**  
Возможно, возникла проблема с VHD, который вы используете для создания рабочей области MED-V.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что вы можете создать виртуальную машину с помощью виртуального жесткого диска для MED-V и что она может быть запущена.

### Событие с кодом 3079

Повторное выполнение подготовки виртуальной машины.

<a href="" id="description"></a>**Описание**  
MED-V пытается подготовить виртуальную машину.

<a href="" id="solution"></a>**Решение**  
Никаких действий не требуется. Начало установки первого раза.

### Событие с кодом 3080

Работа клиента была остановлена при подготовке виртуальной машины.

<a href="" id="description"></a>**Описание**  
При попытке подготовить виртуальную машину была неожиданно остановлена MED-V.

<a href="" id="solution"></a>**Решение**  
Запуск агента хоста MED-V и пошаговое выполнение установки в первый раз

### Событие с кодом 3084

Виртуальная машина является недействительной. При первом запуске программы установки необходимо повторно запустить ее.

<a href="" id="description"></a>**Описание**  
Агент узла MED-V обнаружил проблему с виртуальной машиной.

<a href="" id="solution"></a>**Решение**  
Никаких действий не требуется. Начало установки первого раза.

### Событие с кодом 3099

Не удалось вызвать запуск виртуальной машины.

<a href="" id="description"></a>**Описание**  
Для создания рабочей области MED-V существует потенциальная проблема с VHD, который вы используете.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что вы можете создать виртуальную машину с VHD для MED-V и ее можно открыть.

<a href="" id="solution"></a>**Решение**

### Перенаправление URL-адресов

### Событие с кодом 5005

Не удается получить имя ВМ из конфигурации. не удается запустить браузер гостевой системы.

<a href="" id="description"></a>**Описание**  
Перенаправление URL-адреса не удалось получить из конфигурации имя рабочей области для MED-V. Поэтому он не может информировать Virtual PC для Windows, чтобы открыть перенаправленный URL-адрес в браузере рабочей области MED-V.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что имя рабочей области для MED-V задано и соответствует имени виртуальной машины в каталоге C:\\Users\\ &lt; *User* &gt; \\Virtual Machines. Имя рабочей области для MED-V расположено по адресу HKLM\\SOFTWARE\\Microsoft\\Medv\\v2\\VM\\Name.

Например, если пользователь является "Мэтт", а имя рабочей области — "mattsworkspace", то значение HKLM\\SOFTWARE\\Microsoft\\Medv\\v2\\VM\\Name должно быть "mattsworkspace", а файл с именем C:\\Users\\Matt\\Virtual Machines\\mattsworkspace.vcmx.

### Событие с кодом 5006

Не удалось создать сервер канала.

<a href="" id="description"></a>**Описание**  
Службе перенаправления URL-адресов не удалось создать сервер канала для связи с Internet Explorer.

<a href="" id="solution"></a>**Решение**  
Проверьте журналы системных событий, чтобы попытаться создать файл или ресурс, путь которого начинается примерно так: "\\\\.\\pipe\\MEDVUrlRedirectionPipe\_" и заканчивается именем пользователя и доменным именем пользователя. Если это не пойдет в журнал событий, перезагрузите компьютер.

Данные конфигурации сети узла неправильно отформатированы.

<a href="" id="description"></a>**Описание**

<a href="" id="solution"></a>**Решение**

<a href="" id="description"></a>**Описание**

Эта ошибка может возникать из-за неправильного формата данных или невозможности указать данные в экземпляре CCMNetworkAdapter WMI (инструментарий управления Windows).

<a href="" id="solution"></a>**Решение**

<a href="" id="description"></a>**Описание**

И если он существует, на пути C:\\Users\\Matt\\AppData\\Local\\Microsoft\\MEDV\\v2\\ и всех файлов в нем должны быть разрешения на чтение и запись.

### Гостевая установка исправлений

### Событие с кодом 11001

Сообщение об использовании задачи пробуждения гостя.

<a href="" id="description"></a>**Описание**  
MedvHost.exe с параметром/GuestWakeup был выполнен неправильно или неправильно отформатирован.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что команда выполняется в следующем формате:

Medvhost.exe/GuestWakeup/d: &lt; *duration \ _in \ _minutes* &gt; /v: " &lt; *Рабочая область \ _name* &gt; " где

&lt;*duration _IN \ _minutes* &gt; — количество минут, в течение которых виртуальная машина должна оставаться в спящем (значение по умолчанию — 240).

&lt;*Рабочая область \ _name* &gt; — имя виртуальной машины, которая должна быть пробуждена.

### Событие с кодом 11002

Не удается обновить MED-V — сеть узла не обнаружена.

<a href="" id="description"></a>**Описание**  
Не удалось завершить гостевой патч, так как не обнаружено сетевое соединение с узлом.

<a href="" id="solution"></a>**Решение**  
Подключите узел MED-V к активному сетевому подключению, прежде чем выполнять исправление гостя.

### Событие с кодом 11003

Не удается обновить MED-V — узел не запущен на powerFailed/C, чтобы создать сервер канала.

<a href="" id="description"></a>**Описание**  
Гостевая установка не может быть завершена, так как на нем не работает питание от аккумулятора, а не с кабеля питания.

<a href="" id="solution"></a>**Решение**  
Перед запуском гостевых исправлений Подключите главный компьютер к кабелю питания.

### Событие с кодом 14012

Непредвиденная ошибка, когда права на параметры для папки "папка" &lt; *\ _name* &gt; для &lt; *имени*пользователя &gt; .

<a href="" id="description"></a>**Описание**  
Если при установке служб MED-V не удается установить права и разрешения на доступ к определенным папкам, возникает ошибка.

<a href="" id="solution"></a>**Решение**  
Убедитесь в том, что права администратора указаны в следующих папках:

@"%ProgramData%\\Microsoft\\Medv\\AllUsers"

@"%ProgramData%\\Microsoft\\Medv\\MedvLock"

@"%ProgramData%\\Microsoft\\Medv\\Monitoring"

### Событие с кодом 14013

Произошла непредвиденная ошибка при создании файла блокировки.

<a href="" id="description"></a>**Описание**  
Ошибка возникает, когда MED-V не удается создать файл в папке @ "%ProgramData%\\Microsoft\\Medv\\MedvLock" во время установки.

<a href="" id="solution"></a>**Решение**  
Убедитесь, что права администратора находятся в папке MedvLock.
