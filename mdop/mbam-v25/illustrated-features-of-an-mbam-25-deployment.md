# Иллюстрированные функции развертывания MBAM 2.5


В этой статье описаны отдельные возможности, которые составляют развертывание Microsoft BitLocker и мониторинг (MBAM) 2,5 для следующих топологий.

-   MBAM отдельно

-   Интеграция System Center Configuration Manager

**Важно.**  
Эти функции не представляют рекомендованную архитектуру для развертывания MBAM. Эти сведения можно использовать только в качестве руководства для понимания отдельных функций, которые составляют развертывание MBAM. Рекомендуемая архитектура [MBAM 2,5](high-level-architecture-for-mbam-25.md) для MBAM.



Список поддерживаемых версий программного обеспечения, описанных в этом разделе, можно найти в разделе [Поддерживаемые конфигурации MBAM 2,5](mbam-25-supported-configurations.md).

## <a href="" id="bkmk-standalone"></a> Изолированная топология MBAM


На приведенном ниже рисунке и в таблице описаны возможности изолированной топологии MBAM.

![mbab2\-5](images/mbam2-5-standalonecomponents.png)

|Тип компонента|Описание|База данных|
|-|-|-|
|База данных восстановления|Эта база данных хранит данные для восстановления, собранные из клиентских компьютеров MBAM.|Эта функция настраивается на сервере под управлением Windows Server и поддерживаемом экземпляре SQL Server.|
|База данных соответствия требованиям и аудита|Эта база данных хранит данные о соответствии требованиям, которые используются преимущественно для отчетов, которые размещаются в службах отчетов SQL Server.|Эта функция настраивается на сервере под управлением Windows Server и поддерживаемом экземпляре SQL Server.|
|Отчеты о соответствии и аудите|||
|Веб-служба отчетов|Эта веб-служба обеспечивает связь между веб-сайтом администрирования и мониторинга и экземпляром SQL Server, на котором хранятся данные отчетов.|Этот компонент установлен на сервере под управлением Windows Server.|
|Веб-сайт отчетов (веб-сайт администрирования и мониторинга)|Вы Просматривайте отчеты на веб-сайте администрирования и мониторинга. Отчеты содержат сведения о аудите восстановления и состоянии соответствия требованиям для клиентских компьютеров в вашей организации.|Эта функция настраивается на сервере под управлением Windows Server.|
|Службы SQL Server Reporting Services (SSRS)|Отчеты настраиваются в экземпляре базы данных SSRS. Отчеты можно просматривать непосредственно из служб SSRS или с веб-сайта администрирования и мониторинга.|Эта функция настраивается на сервере под управлением Windows Server и поддерживаемом экземпляре SQL Server, на котором запущены службы SSRS.|
|Сервер самообслуживания|||
|Веб-служба самообслуживания|Эта веб-служба используется клиентом MBAM и веб-сайтом администрирования и мониторинга и порталом самообслуживания для связи с базой данных восстановления.|Этот компонент установлен на компьютере под управлением Windows Server.|
|Самообслуживание на веб-сайте (портал самообслуживания)|С помощью этого веб-сайта конечные пользователи на клиентских компьютерах смогут отдельно входить на веб-сайт, чтобы получить ключ восстановления, если он теряет или забывает пароль BitLocker.|Эта функция настраивается на компьютере под управлением Windows Server.|
|Сервер администрирования и мониторинга|||
|Веб-служба администрирования и мониторинга|Веб-служба мониторинга используется клиентом MBAM и веб-сайтами для связи с базами данных.|Этот компонент установлен на компьютере под управлением Windows Server.|

**Важно.**  
Веб-служба самообслуживания больше не доступна в администрировании и мониторинге Microsoft BitLocker (MBAM) 2,5 с пакетом обновления 1 (SP1), в котором клиент MBAM, веб-сайт администрирования и мониторинга, а также портал самообслуживания прямо взаимодействуют с базой данных восстановления.

**Важно.**  
Веб-служба мониторинга больше не доступна в администрировании и мониторинге Microsoft BitLocker (MBAM) 2,5 с пакетом обновления 1 (SP1), так как клиент MBAM и веб-сайты обмениваются данными непосредственно с базой данных восстановления.
