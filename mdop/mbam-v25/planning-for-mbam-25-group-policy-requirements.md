### Поддержка применяемой политики шифрования с помощью пространства

В MBAM 2,5 с пакетом обновления 1 (SP1) Если вы включите использование шифрования пространства с помощью групповой политики BitLocker, клиент MBAM учитывает его.

Этот параметр групповой политики называется **принудительно использовать тип шифрования диска на дисках операционной системы** и находится в следующем узле GPO: **Параметры конфигурации компьютера** , &gt; **шаблоны** , &gt; **компоненты Windows** , &gt; диски операционной системы с **шифрованием диска BitLocker** &gt; **Operating System Drives**. Если вы включите эту политику и выбрали тип шифрования " **шифрование только**занятого места", MBAM будет соблюдать политику, и BitLocker забудет только шифровать дисковое пространство, используемое на томе.

## Описание параметров групповой политики MBAM


Узел GPO **MBAM для MDOP** состоит из четырех глобальных параметров политики и четырех дочерних узлов GPO: **Управление клиентами**, **стационарные диски**, **диски операционной системы**и **съемные диски**. В следующих разделах описаны и предлагаются параметры для параметров групповой политики MBAM.

**Важно.**  
Не изменяйте параметры групповой политики в разделе " **Шифрование диска BitLocker** " или MBAM будут работать неправильно. MBAM автоматически настраивает параметры на этом узле при настройке параметров в узле **MDOP MBAM (Управление BitLocker)** .

<strong>Важно.</strong><br/><p>Этот параметр групповой политики применяется только на компьютерах с аппаратной конфигурацией UEFI.</p>
</div>
<div>

</div>
<p>Если вы включаете этот параметр политики до включения BitLocker, вы можете настроить компоненты загрузки, которые проверяет доверенный платформенный модуль, прежде чем разблокируйте доступ к диску операционной системы, зашифрованному с помощью BitLocker. Если какое – либо из этих компонентов изменится, когда включена защита BitLocker, доверенный платформенный модуль не освобождает ключ шифрования для разблокировки диска, а на компьютере вместо этого на нем отображается пароль восстановления BitLocker и для разблокировки диска требуется указать его или ключ восстановления.</p>
<p>Если вы отключаете или не настраиваете этот параметр политики, BitLocker использует профиль проверки платформы по умолчанию или профиль проверки платформы, указанный в сценарии настройки.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Сброс данных проверки платформы после восстановления BitLocker</p></td>
<td align="left"><p>Предлагаемая конфигурация: <strong> не настроена</strong></p>
<p>Используйте этот параметр политики, чтобы указать, нужно ли обновлять данные проверки платформы при запуске Windows после восстановления BitLocker.</p>
<p>Если вы включите этот параметр политики, данные проверки платформы будут обновляться при запуске Windows после восстановления BitLocker. Если вы отключаете этот параметр политики, данные проверки платформы не обновляются при запуске Windows после восстановления BitLocker. Если вы не настраиваете этот параметр политики, данные проверки платформы будут обновляться при запуске Windows после восстановления BitLocker.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Использование профиля проверки данных конфигурации с расширенной загрузкой</p></td>
<td align="left"><p>Предлагаемая конфигурация: <strong> не настроена</strong></p>
<p>Этот параметр политики позволяет выбрать определенные параметры данных конфигурации загрузки (BCD) для проверки во время проверки платформы.</p>
<p>Если вы включаете этот параметр политики, вы можете добавлять дополнительные параметры, удалять параметры по умолчанию или и то, и другое. Если вы отключите этот параметр политики, компьютер вернется к профилю BCD, аналогичному профилю BCD по умолчанию, который используется в Windows 7. Если вы не настраиваете этот параметр политики, на компьютере проверяются параметры Windows BCD по умолчанию.</p>
<div class="alert">
<strong>Примечание.</strong><br/><p>Если BitLocker использует безопасную загрузку для платформ и проверки целостности данных конфигурации (BCD) в соответствии с политикой " &quot; Разрешить безопасную загрузку для проверки целостности &quot; ", то &quot; параметр использовать политику проверки данных конфигурации усиленной загрузки &quot; пропускается.</p>
</div>

</tr>

<td align="left"><p>Параметры применения политики шифрования</p></td>
<td align="left"><p>Рекомендуемая конфигурация: <strong> включена</strong></p>
<p>С помощью этого параметра политики можно настроить количество дней, в течение которых пользователи могут откладывать подсвойства с помощью политик MBAM на диске операционной системы. Льготный период начинается при первой обнаружении операционной системы как несоответствия требованиям. По истечении этого льготного периода пользователи не могут откладывать необходимые действия или запрашивать исключение от него.</p>
<p>Если процесс шифрования требует ввода данных пользователем, появится диалоговое окно, в котором пользователи не могут закрыться, пока они не предоставят нужные сведения.</p>
<p>Если вы отключаете или не настраиваете этот параметр, пользователи не будут вынуждены соблюдать политики MBAM.</p>
<p>Если для добавления предохранителя не требуется вмешательство пользователя, шифрование начинается в фоновом режиме по истечении льготного периода.</p></td>
</tr>

<ul>
<li><p><strong>Использовать настраиваемое сообщение </strong> для восстановления: Выберите этот параметр, чтобы включить настраиваемое сообщение в окно восстановления BitLocker с предварительной загрузкой. В <strong> диалоговом окне настраиваемое сообщение о восстановлении </strong> Введите сообщение, которое нужно отобразить. Если вы также хотите указать URL-адрес восстановления, добавьте его в сообщение о том, что вы указали на свое личное восстановление.</p></li>
<li><p><strong>Использовать настраиваемый URL-адрес восстановления </strong> : Выберите этот параметр, чтобы заменить URL-адрес по умолчанию, который отображается на экране восстановления BitLocker с предварительной загрузкой. В <strong> поле настраиваемый URL-адрес восстановления </strong> введите URL-адрес, который нужно отобразить.</p></li>
<li><p><strong>Использовать сообщение о восстановлении по умолчанию и URL-адрес </strong> : Выберите этот параметр, чтобы отобразить сообщение восстановления BitLocker по умолчанию и URL-адрес на экране восстановления BitLocker с предварительной загрузкой. Если вы уже настроили пользовательское сообщение или URL-адрес восстановления и хотите вернуться к сообщению по умолчанию, необходимо включить эту политику и выбрать <strong> параметр использовать сообщение о восстановлении по умолчанию и URL-адрес </strong> .</p></li>
</ul>
