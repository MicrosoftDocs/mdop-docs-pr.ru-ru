# Развертывание MBAM 2,5 в автономной конфигурации

В этой статье приведены пошаговые инструкции по установке администрирования и мониторинга Microsoft BitLocker (MBAM) 2,5 в автономной конфигурации. В этом руководстве мы будем использовать конфигурацию с двумя серверами. Сервер баз данных, на котором работает Microsoft SQL Server 2012, — один из двух серверов. На этом сервере будут размещены базы данных и отчеты MBAM. Дополнительный сервер — это веб-сервер Windows Server 2012, на котором размещаются сервер администрирования и наблюдения и "Самообслуживание портала".

## Подготовительные действия перед установкой серверного программного обеспечения MBAM 2,5

### Шаг 1: Установка и Настройка серверов

Прежде чем приступить к настройке MBAM 2,5, необходимо убедиться, что оба сервера настроены в соответствии с требованиями системы MBAM. Ознакомьтесь с [MBAM минимальными требованиями к системе](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/mbam-25-supported-configurations#-mbam-server-system-requirements)и выберите конфигурацию, которая соответствует этим требованиям. 

#### Шаг 1,1: развертывание предварительных требований для базы данных и сервера отчетов

1. Установка и настройка сервера под управлением операционной системы Windows Server 2008 R2 (или более поздней версии).

2. Установите Windows PowerShell 3,0.

3. Установите Microsoft SQL Server 2008 R2 или более поздней версии, которая включает последний пакет обновления. Если вы устанавливаете новый экземпляр SQL Server для MBAM, убедитесь, что устанавливаемый SQL Server содержит SQL_Latin1_General_CP1_CI_AS параметры сортировки. Вам потребуется установить следующие возможности сервера SQL Server:

    * Ядро СУБД
    * Службы отчетов
    * Клиентские средства подключения
    * Средства управления — завершено

    > [!Note]
    > Кроме того, вы можете установить [функцию шифрования прозрачных данных (TDE) в SQL Server](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/mbam-25-security-considerations).

    Службы SQL Server Reporting Services должны быть установлены и настроены в режиме "машинный", а не в режиме "не настроен" или "SharePoint".

    ![Необходимые возможности SQL Server](images/deploying-MBAM-1.png)

4. Если вы планируете использовать SSL для веб-сайта администрирования и мониторинга, убедитесь, что вы настроили службы SQL Server Reporting Services (SSRS) для использования протокола SSL (Secure Sockets Layer) перед настройкой веб-сайта администрирования и мониторинга. В противном случае функция "отчеты" будет использовать транспорт данных с нешифрованием (HTTP) вместо шифрования (HTTPS).

    Вы можете [настроить соединения SSL](https://docs.microsoft.com/sql/reporting-services/security/configure-ssl-connections-on-a-native-mode-report-server?view=sql-server-2017) на сервере отчетов в собственном режиме, чтобы настроить SSL на сервере отчетов.
    
    > [!Note]
    > Вы можете выполнить руководство по установке SQL Server для соответствующей версии SQL Server, чтобы установить SQL Server. Ниже указаны ссылки.
        > * [SQL Server 2014](https://docs.microsoft.com/sql/sql-server/install/planning-a-sql-server-installation?view=sql-server-2014)
        > * [SQL Server 2012](https://docs.microsoft.com/previous-versions/sql/sql-server-2012/bb500442(v=sql.110))
        > * [SQL Server 2008 R2](https://docs.microsoft.com/previous-versions/sql/sql-server-2012/bb500442(v=sql.110))

5. Убедитесь в том, что после установки SQL Server вы предназначаете учетную запись пользователя в SQL Server, и назначьте следующие разрешения для пользователя, который настроил базу данных MBAM и роли отчетов на сервере базы данных.

    Роли для экземпляра SQL Server:
        
    * dbcreator
    * processadmin

    Права для экземпляра служб SQL Server Reporting Services.
    
    * Создание папок
    * Публикация отчетов

Сервер базы данных готов к настройке ролей MBAM 2,5. Перейдем к следующему серверу.

### Действие 2: создание пользователей и групп в доменных службах Active Directory
 
В рамках предварительных требований вы должны определить определенные роли и учетные записи, которые используются в MBAM, чтобы обеспечить безопасность и получать права доступа к определенным серверам и функциям, например к базам данных, работающим на экземпляре SQL Server и веб-приложениям, которые выполняются на сервере администрирования и мониторинга.

Создайте следующие группы и пользователей в Active Directory. (Вы можете использовать любое имя для групп и пользователей.) Пользователям не нужно больше прав пользователя. Достаточно учетной записи пользователя домена. Вы должны указать имена этих групп во время настройки MBAM 2,5:

* **MBAMAppPool**  

  **Тип**: пользователь домена

  **Описание**: пользователь домена, у которого есть разрешение на чтение и запись в базу данных соответствия и аудита, а также база данных восстановления, позволяющая веб-приложениям получать доступ к данным и отчетам в этих базах данных. Она также будет использоваться пулом приложений для веб-приложений.

  **Роли учетной записи (во время настройки MBAM)**:

  1. Учетная запись домена пула приложений веб-службы

  2. Учетная запись и аудит для базы данных и базы данных восстановления для чтения и записи в отчетах

* **MBAMROUser**

  **Тип**: пользователь домена

  **Описание**: пользователь домена, имеющий доступ к базе данных "соответствие" и "аудит только для чтения" для обеспечения доступа к данным о соответствии и аудиту в этой базе данных. Она также будет учетной записью пользователя домена, которую локальный экземпляр служб SQL Server Reporting Services использует для доступа к базе данных соответствия и аудита.

  **Роли учетной записи (во время настройки MBAM)**:

  1. Соответствие требованиям к базам данных и аудит для отчетов, предназначенных только для чтения

  2. Учетная запись пользователя домена базы данных соответствия требованиям и аудита

* **MBAMAdvHelpDsk**

  **Тип**: группа домена

  **Описание**: Группа "Пользователи" расширенной поддержки MBAM: Группа "Пользователи домена", у участников которой есть доступ ко всем областям веб-сайта администрирования и мониторинга. Пользователи с этой ролью должны вводить только ключ восстановления, а не домен пользователя и имя пользователя, когда они помогают пользователям восстановить свои диски. Если пользователь является членом обеих групп "Пользователи службы поддержки MBAM" и "Пользователи расширенной справочной службы", то разрешения группы "Пользователи расширенной поддержки" переопределяют разрешения для группы поддержки MBAM.

  **Роли учетной записи (в ходе настройки MBAM)**: MBAM "Расширенные пользователи службы поддержки"

* **MBAMHelpDsk**

  **Тип**: группа домена

  **Описание**: пользователи службы поддержки MBAM: группа пользователей домена, у участников которой есть доступ к разделу Управление доверенными платформенными модулями и для восстановления на веб-сайте MBAM администрирования и мониторинга. Пользователи, которым назначена эта роль, должны заполнить все поля при использовании любого из этих параметров. Сюда входит домен и имя учетной записи пользователя.

  **Роли учетной записи (во время настройки MBAM)**: пользователи службы поддержки MBAM

* **MBAMRUGrp**

  **Тип**: группа домена

  **Описание**: группа пользователей домена, участники которой имеют доступ к отчетам только для чтения в области "отчеты" на веб-сайте администрирования и мониторинга.

  **Роли учетной записи (во время настройки MBAM)**:

  1. Отчетная Группа доступа к домену, доступная только для чтения

  2. Группа "Пользователи отчетов MBAM"

### Шаг 3 (необязательно): Настройка и Установка SSL-сертификата на сервере администрирования и мониторинга

Несмотря на то что это не является обязательным, мы настоятельно рекомендуем использовать сертификат для обеспечения безопасности связи между клиентом MBAM и веб-сайтом администрирования и мониторинга и веб-сайтами портала самообслуживания. Мы не рекомендуем использовать самозаверяющие сертификаты из-за очевидных соображений безопасности. Мы рекомендуем использовать сертификат типа веб-сервера в доверенном центре сертификации. Для этого вы можете обратиться к разделу "использование сертификата, утвержденного центром сертификации" из [статьи KB 2754259](https://support.microsoft.com/help/2754259).

После выдачи сертификата вы должны добавить сертификат в личное хранилище сервера администрирования и мониторинга. Чтобы добавить сертификат, откройте хранилище сертификатов на локальном компьютере. Для этого выполните указанные ниже действия.

   ![Выбор](images/deploying-MBAM-4.png)

   ![Окно добавления и удаления оснасток](images/deploying-MBAM-5.png)

### Шаг 4: Настройка сертификата SSL для веб-сервера MBAM

Если вы используете SSL-связь между клиентом и сервером, убедитесь в том, что сертификат содержит идентификаторы OID расширенного использования ключа (1.3.6.1.5.5.7.3.1) и (1.3.6.1.5.5.7.3.2). То есть следует убедиться, что проверка подлинности сервера и клиента добавлена.

Если при попытке просмотреть URL-адреса службы вы получаете сообщение об ошибке с сертификатом, вы используете сертификат, выданный другим именем, или при просмотре с использованием неправильного URL-адреса.

Несмотря на то, что браузер может запросить сообщение об ошибке сертификата, но продолжить работу, веб-служба MBAM не будет игнорировать ошибки сертификата и блокировать подключение. Вы увидите ошибки, связанные с сертификатами, в журнале событий администратора MBAM клиента MBAM. Если вы используете псевдоним для подключения к серверу администрирования и наблюдения, вы должны выдать сертификат для имени псевдонима. Таким же именем субъекта сертификата должен быть имя псевдонима, а DNS-имя локального сервера нужно добавить в поле " **дополнительное имя субъекта** " сертификата.

Пример.

Если для виртуального имени задано значение "bitlocker.contoso.com", а в качестве имени сервера администрирования и мониторинга MBAM — "adminserver.contoso.com", то сертификат должен быть выдан bitlocker.contoso.com (имя субъекта), и adminserver.contoso.com следует добавить в поле **дополнительного имени субъекта** сертификата.

Аналогичным образом, если на компьютере установлено несколько серверов администрирования и мониторинга для балансировки нагрузки с помощью подсистемы балансировки нагрузки, сертификат SSL следует выдавать в виртуальном имени. Это значит, что поле имени субъекта сертификата должно содержать виртуальное имя, а имена всех локальных серверов должны быть добавлены в поле " **дополнительное имя субъекта** " сертификата.

Пример.

Если виртуальным именем является "bitlocker.contoso.com", а серверы — "adminserver1.contoso.com" и "adminiserver2.contoso.com", то сертификат должен быть выдан bitlocker.contoso.com (имя субъекта) и adminserver1.contoso.com, и adminiserver2.contoso.com следует добавить в поле " **дополнительное имя субъекта** " сертификата.

Инструкции по настройке связи SSL с помощью MBAM описаны в следующей статье базы знаний Майкрософт: [KB 2754259](https://support.microsoft.com/help/2754259).

### Действие 5: Зарегистрируйте SPN для учетной записи пула приложений и настройте ограниченное делегирование.

> [!Note]
> Ограниченное делегирование требуется только для 2,5 и не требуется для 2,5 с пакетом обновления 1 (SP1) и более поздних версий.

Чтобы разрешить серверам MBAM возможность проверки подлинности связи на веб-сайте администрирования и мониторинга и на портале самообслуживания, необходимо зарегистрировать имя участника-службы (SPN) для имени узла в учетной записи домена, которую вы используете для работы с этим пулом приложений. В следующей статье содержатся пошаговые инструкции по регистрации SPN: [Планирование безопасности веб-сайтов MBAM](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/planning-how-to-secure-the-mbam-websites)

После настройки имени участника-службы необходимо настроить ограниченное делегирование для SPN. Для этого выполните указанные ниже действия.

1. Перейдите в службу каталогов Active Directory и найдите учетные данные пула приложений, настроенные для веб-сайтов MBAM, на предыдущих шагах.

2. Щелкните учетные данные правой кнопкой мыши и выберите пункт **Свойства**.

3. Откройте вкладку **Делегирование** .

4. Выберите параметр для проверки подлинности Kerberos.

5. Нажмите кнопку **Обзор**и перейдите к учетным данным пула приложений еще раз. Затем вы увидите все SPN, настроенные на учетной записи группы приложений. (Имя SPN должно иметь вид "HTTP/BitLocker. FQDN. com"). Выделит имя участника-службы, совпадающее с именем узла, которое вы указали при установке MBAM.

6. Нажмите кнопку **ОК**.

Теперь вы хорошо подходящими для всех необходимых компонентов. На следующих шагах предстоит установить программное обеспечение MBAM на серверы и настроить его.

### Шаг 10: Настройка портала с самостоятельным сервером, если клиентские компьютеры не могут получить доступ к сети CDN 

Определите, имеют ли клиентские компьютеры доступ к сети доставки содержимого (CDN) Microsoft AJAX. Сеть CDN предоставляет порталу самообслуживания, доступ к которому требуется для определенных файлов JavaScript. Если вы не настраиваете портал самообслуживания, когда клиентские компьютеры не могут получить доступ к сети CDN, будет отображаться только название компании и учетная запись, под которой вошел пользователь. Сообщение об ошибке не выводится.

Выполните одно из следующих действий.

* Если клиентские компьютеры имеют доступ к сети CDN, ничего делать не нужно. Настройка портала самообслуживания завершена.

* Если клиентские компьютеры не имеют доступа к сети CDN, выполните действия, описанные в разделе [Настройка портала самообслуживания, когда клиентские компьютеры не могут получить доступ к Интернету для доставки содержимого](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/how-to-configure-the-self-service-portal-when-client-computers-cannot-access-the-microsoft-content-delivery-network).

### Шаг 11: Проверка конфигурации компонентов сервера MBAM 2,5 

Чтобы проверить развертывание сервера MBAM на использование автономной топологии, выполните указанные ниже действия.

   > [!Note]

### Шаг 12: Настройка шаблонов групповой политики MBAM

Чтобы развернуть MBAM, необходимо задать параметры групповой политики, определяющие параметры реализации MBAM для шифрования диска BitLocker. Для выполнения этой задачи необходимо скопировать шаблоны групповой политики MBAM на сервер или рабочую станцию, на которых может выполняться консоль управления групповыми политиками (GPMC) или расширенное управление групповыми политиками, а затем изменить параметры.

> [!Important]
> Не изменяйте параметры групповой политики в узле **шифрования диска BitLocker** или MBAM будут работать неправильно. При настройке параметров групповой политики на узле **MDOP MBAM (Управление BitLocker)** MBAM автоматически настраивает параметры **шифрования диска для BitLocker** .
 
#### Копирование шаблонов групповой политики MBAM 2,5

Перед установкой клиента MBAM необходимо скопировать MBAM-специфические объекты групповой политики (GPO) на рабочую станцию управления. Эти GPO определяют параметры реализации MBAM для BitLocker. Вы можете скопировать шаблоны групповой политики на любой сервер или рабочую станцию, которые поддерживаются на сервере или клиентском компьютере под управлением Windows и которые могут запускать консоль управления групповыми политиками (GPMC) или расширенное управление групповыми политиками.

Дополнительные сведения можно найти [в разделе Копирование шаблонов групповой политики MBAM 2,5](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/copying-the-mbam-25-group-policy-templates).
 
#### Изменение параметров объекта групповой политики MBAM 2,5

После создания необходимых GPO необходимо развернуть параметры групповой политики MBAM на клиентских компьютерах своей организации. Для просмотра и создания объектов групповой политики необходимо установить консоль управления групповыми политиками (GPMC) или расширенное управление групповыми политиками.

Дополнительные сведения можно найти [в разделе Изменение параметров групповой политики MBAM 2,5](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/editing-the-mbam-25-group-policy-settings) и [Планирование требований к групповой политике MBAM 2,5](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/planning-for-mbam-25-group-policy-requirements).
 
### Шаг 13: развертывание клиента MBAM 2,5

В зависимости от того, как вы развертываете клиентское программное обеспечение для администрирования Microsoft BitLocker и наблюдения за ним, вы можете включить BitLocker на компьютере в вашей организации либо перед получением пользователем компьютера, либо позже, настроив групповую политику и развернув клиентское программное обеспечение MBAM с помощью корпоративной системы развертывания программного обеспечения.
 
#### Развертывание клиента MBAM на настольных или переносных компьютерах

После того как вы настроили параметры групповой политики, вы можете развернуть установочные файлы установщика Windows на целевые компьютеры, используя корпоративный продукт системы развертывания программного обеспечения, например Microsoft System Center 2012 Configuration Manager или доменные службы Active Directory (AD DS). Вы можете использовать либо 32-разрядный, либо 64-разрядный MbamClientSetup.exe, либо файлы 32 или 64-bit MBAMClient.msi. Они предоставляются вместе с программным обеспечением клиента MBAM.

Дополнительные сведения можно найти в разделе [развертывание клиента MBAM на настольных компьютерах и ноутбуках](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/how-to-deploy-the-mbam-client-to-desktop-or-laptop-computers-mbam-25).
 
#### Развертывание клиента MBAM в рамках развертывания Windows

В организациях, в которых устанавливаются и изменяются компьютеры, вы можете установить клиент MBAM для управления шифрованием диска BitLocker на каждом компьютере до того, как на него будут записываться пользовательские данные. Преимущество этого процесса состоит в том, что каждый компьютер затем соответствует требованиям, предъявляемым к BitLocker. Этот метод не полагается на действия пользователя, так как администратор уже зашифровал компьютер. Ключевым предположением для этого сценария является то, что политика Организации должна устанавливать корпоративный образ Windows до того, как компьютер будет доставлен пользователю. Если в параметрах групповой политики требуется ПИН-код, пользователи получат запрос на установку PIN-кода после того, как они получит политику.

Дополнительные сведения можно найти в разделе [развертывание клиента MBAM в рамках развертывания Windows](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/how-to-enable-bitlocker-by-using-mbam-as-part-of-a-windows-deploymentmbam-25).
 
#### Развертывание клиента MBAM с помощью командной строки

Дополнительные сведения можно найти в разделе [развертывание клиента MBAM с помощью командной строки](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/how-to-deploy-the-mbam-client-by-using-a-command-line).

После того как вы закончите работу по развертыванию, вы должны просмотреть следующие журналы и определить, успешно ли сообщается о них в базу данных MBAM.

## Вопросы и ответы

### Создание серверов IIS с балансировкой нагрузки

* SPN должны быть зарегистрированы только в понятном имени (например,: bitlocker.corp.net) и не должны регистрироваться на индивидуальные серверы IIS.

* При использовании сертификата сертификат должен иметь оба имени FQDN и NetBIOS, введенные в поле " **дополнительное имя субъекта** ", для всех серверов IIS в группе "Балансировка нагрузки", а также как понятное имя (например: BitLocker.Corp.NET). В противном случае сертификат будет считаться ненадежным для просмотра при просмотре адресов, распределенных по загрузке.

Дополнительные сведения можно найти в разделе [Балансировка сетевой нагрузки служб IIS](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/planning-for-mbam-25-high-availability#a-href-idbkmk-load-balanceaiis-network-load-balancing) и [Регистрация SPN для учетной записи пула приложений](https://docs.microsoft.com/microsoft-desktop-optimization-pack/mbam-v25/planning-how-to-secure-the-mbam-websites#registering-spns-for-the-application-pool-account).

### Настройка сертификата

* Вам потребуется два сертификата. Для SQL Server используется один сертификат, а другой — для IIS. Они должны быть установлены до начала установки MBAM.

* Рекомендуется использовать установщик для добавления сертификата в конфигурацию IIS вместо того, чтобы вручную вносить изменения в файл web.config.

* Сертификат не будет принят конфигуратором MBAM, если поле «Кому выдан» сертификата не совпадает с именем сервера. В этом случае вы временно создаете самозаверяющий сертификат на консоли IIS и используете его в конфигураторе. Это сделает Nsure, что веб-приложения установлены для SSL и HTTPS. После этого вы можете изменить сертификат на один из привязок IIS для веб-сайта MBAM.

### Требования к разрешениям SQL для установки

Создайте учетную запись для пула приложений MBAM и предоставьте ей только разрешения для упрощения, публичного доступа и создателя DBCreator.

Более подробную информацию вы видите в разделе [Конфигурация базы данных MBAM — минимальные разрешения](https://blogs.technet.microsoft.com/dubaisec/2016/02/02/mbam-database-configuration-minimum-permissions/) .

> [!Note]
> * В некоторых ситуациях для начальной установки и обновления требуются дополнительные разрешения.
> * Используйте учетную запись, которая содержит временное SA для установки.
> * Не запускайте конфигуратор в контексте учетной записи пользователя (Запуск от имени), у которого нет достаточных разрешений на внесение изменений в SQL Server, так как это приводит к ошибкам при установке.
> * Необходимо войти в систему с помощью учетной записи, имеющей разрешения на доступ к SQL Server. Только базы данных SQL Server могут быть созданы или обновлены при удаленном запуске конфигуратора MBAM. Для сервера служб SSRS необходимо установить MBAM и запустить конфигуратор локально, чтобы установить или обновить отчеты SSRS для MBAM.

### Разрешение, требуемое для регистрации SPN

Для учетной записи, используемой для установки портала IIS, должны быть указаны разрешения на запись и на запись проверенных имен участников-служб. Без этих разрешений при установке будет выдано предупреждение о том, что не удается зарегистрировать SPN.

> [!Note]
> Это предупреждающее сообщение будет появляться дважды. Это не означает, что для SPN должно быть зарегистрировано два объекта.

Дополнительные сведения можно найти в разделе Выполнение [программы установки MBAM завершается сбоем с сообщением об ошибке "регистрация SPN отложено"](https://support.microsoft.com/help/2754138/).
