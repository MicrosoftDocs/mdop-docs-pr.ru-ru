## Известные проблемы с пакетом обновления 4 для Application Virtualization 5,0 SP2


### Пакеты перестают работать после удаления исправления 4 для Application Virtualization 5,0 с пакетом обновления 2 (SP2)

Пакеты, опубликованные в пакете исправлений 4 для Application Virtualization 5,0, перестают работать, когда удаляется пакет исправлений 4 для Application 5,0 Virtualization с пакетом обновления 2 (SP2).

СМЫСЛА

Если папка находится в следующей папке, ее необходимо удалить.

**% LocalAppData%**  \\  **Microsoft**  \\  Службы **AppV**  \\  **Client (клиент**  \\  ) **VFS**  \\  ** &lt; идентификатор &gt; пакета** для каждого опубликованного пакета.

**Примечание**  Вы должны иметь права на повышение прав для удаления этой папки.

 

Чтобы использовать сценарий, для каждой учетной записи пользователя на компьютере и для каждого идентификатора пакета, опубликованного после установки исправления 4 для Application Virtualization 5,0 с пакетом обновления 2 (SP2), выполните указанные ниже действия.

`Rd /s /q “%systemdrive%\users\[UserName]\AppData\Local\Microsoft\AppV\Client\VFS\[Package ID]`

-   Сочетания клавиш будут сохраняться в сеансах пользователей даже после удаления папки из каталога, описанного в предыдущем разделе, поэтому вы можете щелкнуть ярлык, чтобы запустить приложение еще раз. Повторно опубликовать приложение не требуется.

-   Эта проблема возникает для обоих опубликованных пакетных и глобально опубликованных пакетов, например Microsoft Office2013. Папку необходимо удалить для обоих типов пакетов.

-   Не нужно удалять папку VFS в данных перемещаемого приложения (**% AppData%**). Только **% LocalAppData%** должен быть удален.

### Интеграция с Microsoft Office указывает на неправильное расположение в файловой системе

Интеграция с Microsoft Office указывает на неправильное расположение файловой системы (Groove.exe сообщение об ошибке).

СМЫСЛА

Воспользуйтесь одним из указанных ниже способов.

1.  После обновления удалите ярлык из папки "начало".

2.  Измените сочетание клавиш в начальной папке с помощью сценария.

3.  С помощью файла конфигурации развертывания можно указать целевой объект ярлыка для корня интеграции.

### <a href="" id="-------------hotfix-package-4-for-application-virtualization-5-0-sp2-installer-can-take-a-long-time"></a> Пакет исправлений 4 (SP4) для Application Virtualization 5,0 может занять много времени

Пакет исправлений 4 (SP2) для Application Virtualization 5,0 может занять много времени в зависимости от того, сколько файлов хранится в существующем кэше пакетов.

Обновление связанных дескрипторов безопасности пакетов в пакете исправлений 4 для Application Virtualization 5,0 с пакетом обновления 2 (SP2) значительно сказывается на том, сколько времени займет установка. Ранее установка была стандартной в течение длительного времени. Тем не менее, теперь он зависит от количества файлов, размещенных в кэше пакетов.

ВРЕМЕННОе решение: нет

### Удаление исправления пакета 4 для Application Virtualization 5,0 SP2 завершается сбоем, если используется пакет с JIT-V.

Если вы установили пакет исправлений 4 для Application Virtualization 5,0, а затем пытаетесь удалить исправление при использовании JIT-V, операция завершится сбоем, если выполняются все указанные ниже условия.

-   Вы установили с помощью файла установщика Windows (MSI), а затем примените обновления с помощью файла исправления Microsoft Installer (MSP).

-   Вы пытаетесь удалить обновление с помощью элемента "Установка и удаление программ" на панели управления.

-   На компьютере выполняется пакет с поддержкой JIT-V.

ВРЕМЕННОе решение: выполните указанные ниже действия.

1.  Откройте Windows PowerShell и выполните следующие команды:

    -   **Import-Module appvclient**

    -   **Get-AppvClientPackage | Остановить-AppvClientPackage**

2.  Удалите обновление с помощью компонента "Установка и удаление программ".

## Известные проблемы с приложением-V 5,0 SP2


### <a href="" id="bkmk-folderredirection"></a>Перенаправление папок клиента App-V иногда не может переносить файлы из% AppData% в% LocalAppData%

Если% AppData% — общая сетевая папка, настроенная для перенаправления папок, изменения, внесенные конечными пользователями в AppData (роуминг), могут быть потеряны при переключении на компьютеры или при отключении локальной папки AppData при выходе из нее и повторном входе. Эта ошибка возникает из-за того, что раздел реестра (AppDataTime), который указывает последнюю известную отправку, выходит из синхронизации с локальной кэшированной AppData.

ВРЕМЕННОе решение: вручную удалите следующий раздел реестра для каждого пакета, когда пользователь входит в систему или выключается.

``` syntax
HKCU\Software\Microsoft\AppV\Client\Packages\<PACKAGE_GUID>\AppDataTime
```

При первом запуске приложения конечными пользователями в пакете после входа в приложение App-V принудительно загружает сжатый% AppData%, даже если% LocalAppData% уже не устарел.

### <a href="" id="-------------app-v-5-0-service-pack-2--app-v-5-0-sp2--does-not-include-a-new-version-of-the-app-v-server"></a> В App-V 5,0 с пакетом обновления 2 (App-V 5,0 SP2) не входит новая версия сервера App-V.

Приложение App-V 5.0 SP2 не содержит новую версию сервера App-V. Если вы развертываете клиенты App-V 5,0 SP2 под управлением Windows 8,1 в своей среде и планируете управлять ими с помощью инфраструктуры App-V, необходимо установить [пакет исправлений 2 для Microsoft Application Virtualization 5,0 с пакетом обновления 1](https://go.microsoft.com/fwlink/?LinkId=386634)(SP1). (https://go.microsoft.com/fwlink/?LinkId=386634)

Если вы работаете с клиентами App-V 5,0 SP2 и управляете ими с помощью любого из указанных ниже способов, обновление клиента не требуется.

-   Автономный режим.

-   Configuration Manager;

-   Сторонние ESD.

Клиент App-V 5,0 с пакетом обновления 2 (SP2) полностью совместим с Windows 8,1

ВРЕМЕННОе решение: нет.
