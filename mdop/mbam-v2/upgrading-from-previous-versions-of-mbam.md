# Обновление с предыдущих версий MBAM


Вы можете обновить администрирование и мониторинг Microsoft BitLocker (MBAM) до MBAM 2.0 с изолированной топологией или топологией Configuration Manager, выполнив указанные ниже действия.

-   **Замена сервера вручную** — чтобы обновить сервер MBAM, вручную удалите MBAM с помощью установщика или панели управления, а затем установите инфраструктуру MBAM 2.0. Вам не нужно удалять базы данных. После удаления сервера MBAM 1,0 базы данных MBAM остаются без изменений. Если указаны те же базы данных, что и MBAM 1,0, установка MBAM 2.0 сохраняет данные MBAM 1,0 в базах данных и преобразует базы данных в MBAM 2.0.

-   **Обновление распределенного клиента** — если вы используете автономную топологию MBAM, вы можете сразу обновить клиенты MBAM после установки инфраструктуры сервера MBAM 2,0. Сервер MBAM 2,0 определяет версию существующего клиента и выполняет необходимые действия по обновлению до клиента 2,0.

    После обновления инфраструктуры сервера MBAM 2,0 клиенты MBAM 1,0 продолжают сообщать об этом на сервер MBAM 2,0, использующий данные для восстановления, но соответствие требованиям будет основываться на политиках в MBAM 1,0. Чтобы клиентские компьютеры MBAM на соответствие требованиям политик MBAM 2,0, необходимо обновить клиенты до версии 2,0. Вы можете обновить клиентские компьютеры до MBAM 2,0 без удаления прежнего клиента, после чего клиент начнет применять и сообщать о политиках MBAM 2,0.

    Если вы используете MBAM с Configuration Manager, вы должны обновить клиенты MBAM 1,0 до MBAM 2,0.

## Обновление MBAM из двух-серверной архитектуры


Выполните указанные ниже инструкции, чтобы обновить предыдущую версию MBAM при использовании архитектуры из двух серверов, где на одном сервере размещены компоненты Microsoft SQL Server, и на другом сервере размещены веб-сайты и службы.

**Обновление MBAM с помощью архитектуры из двух серверов**

1.  На сервере с возможностями SQL Server на панели управления выберите пункт **программы и компоненты**, а затем удалите **Администрирование и мониторинг Microsoft BitLocker**. База данных восстановления и база данных соответствия и аудита остаются без изменений.

2.  Запустите **MBAMSetup.exe** для версии MBAM 2,0, при необходимости выберите **программу улучшения качества программного**обеспечения, а затем нажмите кнопку **начать**.

3.  Прочтите и подтвердите условия лицензионного соглашения на использование программного обеспечения корпорации Майкрософт, а затем нажмите кнопку **Далее** , чтобы продолжить установку.

4.  На странице **Выбор топологии** выберите **изолированную** топологию **интеграции с System Center Configuration Manager** и нажмите кнопку **Далее**.

5.  На странице **Выбор устанавливаемых компонентов** снимите флажок **сервер самообслуживания** , сервер **администрирования и мониторинг** и нажмите кнопку **Далее**.

6.  Дождитесь завершения проверок готовности к установке и нажмите кнопку **Далее**. Если обнаружен отсутствующий предварительный компонент, устраните недостающие необходимые условия и нажмите кнопку **проверить необходимые компоненты еще раз**.

7.  На странице **Предоставление учетной записи, используемой для доступа к базам данных MBAM** укажите имя компьютера для сервера, на котором будут размещены сайты и службы, и нажмите кнопку **Далее**.

8.  На странице **Настройка базы данных восстановления** укажите имя экземпляра SQL Server и имя базы данных, в которой будут храниться данные для восстановления. Кроме того, необходимо указать, где будут храниться файлы базы данных и данные журнала.

9.  Чтобы продолжить, нажмите **Далее**.

10. На странице **Настройка базы данных соответствия и аудита** укажите имя экземпляра SQL Server и имя базы данных, в которой будут храниться сведения о соответствии и аудите.

11. Чтобы продолжить, нажмите **Далее**.

12. На странице " **Настройка отчетов о соответствии и аудите** " укажите экземпляр служб SQL Server Reporting Services, на котором будут установлены отчеты о соответствии и аудите, а также укажите учетную запись пользователя домена и пароль для доступа к базе данных "соответствие требованиям" и "Аудит". Настройка пароля для этой учетной записи бессрочна. Учетная запись пользователя может получать доступ ко всем данным, доступным для группы "Пользователи отчетов MBAM".

13. Чтобы продолжить, нажмите **Далее**.

14. Укажите, следует ли использовать обновления Microsoft, чтобы обеспечить безопасность компьютера, и нажмите кнопку **Далее**. При этом автоматические обновления в Windows не активируются. Если ранее вы выбрали использование центра обновления Майкрософт для этого продукта или другого продукта, страница центра обновления Майкрософт не отображается.

15. На странице **Сводка по установке** просмотрите компоненты, которые будут установлены, и нажмите кнопку **установить** , чтобы начать установку.

**Удаление компонентов сервера администрирования и мониторинга и завершение обновления**

1.  На компьютере, на котором размещены компоненты сервера администрирования и мониторинга, на панели управления выберите пункт **программы и компоненты**и удалите MBAM, чтобы удалить ранее установленные веб-сайты и службы.

2.  Запустите **MBAMSetup.exe** для версии 2,0, при необходимости выберите **программу улучшения качества программного**обеспечения, а затем нажмите кнопку **начать**.

3.  Прочтите и подтвердите условия лицензионного соглашения на использование программного обеспечения корпорации Майкрософт, а затем нажмите кнопку **Далее** , чтобы продолжить установку.

4.  На странице **Выбор топологии** выберите **изолированную** топологию **интеграции с System Center Configuration Manager** и нажмите кнопку **Далее**.

5.  На странице **выберите компоненты для установки** снимите флажки **база данных восстановления** , **базы данных соответствия требованиям** и функции проверки соответствия и **аудита** , а затем нажмите кнопку **Далее**.

6.  Дождитесь завершения проверок готовности к установке и нажмите кнопку **Далее**. Если обнаружен отсутствующий предварительный компонент, сначала устраните недостающие предварительные требования, а затем нажмите кнопку **проверить необходимые компоненты еще раз**.

7.  На странице **Настройка безопасности сетевых подключений** укажите, следует ли использовать шифрование SSL для веб-сайтов и служб. Если вы решили зашифровать связь, выберите сертификат центра сертификации (ЦС), который будет использоваться для шифрования.

    **Примечание**  Прежде чем этот шаг можно будет выбрать на этой странице, вы должны создать сертификат, прежде чем это сделать.

     

8.  На странице **Настройка расположения базы данных о состоянии соответствия требованиям** укажите имя экземпляра SQL Server и имя базы данных, в которой хранятся данные о соответствии и аудите. Кроме того, необходимо указать, где будут храниться файлы базы данных и данные журнала.

9.  Чтобы продолжить, нажмите **Далее**.

10. В разделе **Настройка расположения на странице базы данных восстановления** укажите имя экземпляра SQL Server и имя базы данных, в которой хранятся данные для восстановления.

11. Чтобы продолжить, нажмите **Далее**.

12. На странице **Настройка отчетов о соответствии и аудите** введите URL-адрес экземпляра отчета, настроенного на другом сервере. С помощью кнопки " **тест** " убедитесь, что сайт доступен.

13. Чтобы продолжить, нажмите **Далее**.

14. На странице **Настройка портала самообслуживания** введите номер порта, имя узла, имя виртуального каталога и путь установки для портала самообслуживания.

    **Примечание**  Номер порта, который вы указали, должен быть неиспользуемым номеру на сервере администрирования и мониторинга, если не указано уникальное имя заголовка узла.

     

15. На странице **Настройка сервера администрирования и мониторинга** укажите нужный виртуальный каталог для веб-сайта службы поддержки.

16. Укажите, следует ли использовать обновления Microsoft, чтобы обеспечить безопасность компьютера, и нажмите кнопку **Далее**. На этом этапе не включается функция автоматического обновления в Windows. Если ранее вы выбрали использование центра обновления Майкрософт для этого продукта или другого продукта, страница центра обновления Майкрософт не отображается.

17. На странице **Сводка по установке** просмотрите компоненты, которые будут установлены, и нажмите кнопку **установить** , чтобы начать установку.

18. Чтобы убедиться в том, что обновление прошло успешно, убедитесь, что вы можете подключиться к каждому сайту с другого компьютера домена.

## Обновление клиента MBAM на компьютерах конечных пользователей


Чтобы обновить компьютеры конечных пользователей до клиента MBAM 2,0, запустите **MbamClientSetup.exe** на каждом клиентском компьютере. Установщик автоматически обновляет клиент для клиента MBAM 2,0. Вы можете установить клиент MBAM через электронную систему распространения программного обеспечения, такие инструменты, как доменные службы Active Directory или System Center Configuration Manager.

Чтобы проверить обновление клиента, выполните указанные ниже действия.

1.  Дождитесь завершения настроенного цикла создания отчета, а затем запустите **SQL Server Management Studio** на компьютере SQL Server.

2.  На компьютере SQL Server запустите **Microsoft SQL Server Management Studio**.

3.  Убедитесь, что в таблице **RecoveryAndHardwareCore.** Computers указана строка, в которой указано имя компьютера конечного пользователя.
