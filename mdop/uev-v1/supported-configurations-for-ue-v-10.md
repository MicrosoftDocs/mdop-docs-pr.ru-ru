### <a href="" id="requirements-for-the-offline-files-feature-"></a>Требования для функции автономных файлов

Агент UE-V Agent может синхронизировать параметры пользователей для компьютеров, которые не подключены к корпоративной сети, например ноутбук или компьютеры, расположенные в удаленных офисах, а также компьютеры, которые всегда подключены к корпоративной сети, такие как серверы Windows, на которых размещаются сеансы связи с виртуальными рабочими столами (VDI).

Настройка по умолчанию UE-V использует функцию автономных файлов Windows для синхронизации параметров. Автономные файлы гарантируют доступность параметров пользователя, даже когда компьютер выходит из корпоративной сети. Любые изменения, внесенные в параметры, автоматически синхронизируются с расположением хранилища параметров при повторной установке подключения к корпоративной сети. Автономные файлы также гарантируют, что параметры пользователя доступны для компьютеров, расположенных в удаленном офисе с медленным или ограниченным подключением.

Чтобы синхронизировать параметры для компьютеров, которые периодически оставляют корпоративную сеть, для начала развертывания агента UE-V необходимо включить и запустить функцию автономных файлов. Функция "автономные файлы" включена по умолчанию в Windows7. Эта функция отключена по умолчанию в Windows Server2008R2, Windows Server 2012 и Windows 8. Если функция автономных файлов не включена, синхронизация параметров UE-V завершается сбоем.

-   **Windows 7;**

    Функция "автономные файлы" включена по умолчанию в Windows 7. При необходимости можно включить автономные файлы, выполнив следующую команду в командной строке с повышенными привилегиями:

    ``` syntax
    sc config CscService start=auto
    ```

-   **Windows 8**

    Функция автономных файлов по умолчанию отключена в версии Windows 8. Автономные файлы можно включить в Windows 8, выполнив следующую команду в командной строке с повышенными привилегиями:

    ``` syntax
    sc config CscService start=auto
    ```

-   **Windows Server 2008 R2 и Windows Server 2012**

    Функция "автономные файлы" не установлена по умолчанию в Windows Server 2008 R2 или Windows Server 2012. Чтобы включить функцию автономных файлов, необходимо установить пакет возможностей для рабочего стола. Это необязательный серверный компонент, который включает функцию "автономные файлы". После установки запустите функцию автономных файлов с помощью следующих команд в командной строке с повышенными привилегиями:

    ``` syntax
    sc config csc start= system
    ```

    ``` syntax
    sc config cscservice start= auto
    ```

Чтобы начать синхронизацию параметров, необходимо перезагрузить компьютер.

### Синхронизация для компьютеров с постоянно доступными подключениями

При использовании UE-V на компьютерах, которые всегда подключены к корпоративной сети, например на компьютере с Windows Server, на котором размещаются сеансы VDI, автономные файлы должны быть отключены.

Если агент UE-V настроен для синхронизации параметров без использования автономных файлов, сервер хранилища параметров рассматривается как Стандартная сетевая общий доступ. Параметры синхронизируются, когда сеть доступна. В этой конфигурации агент UE-V можно настроить таким образом, чтобы он мог получать уведомления о задержке импорта параметров приложения.

Если функция автономных файлов не используется, необходимо отключить поведение UE-V по умолчанию до или во время развертывания агента UE-V. Чтобы отключить автономные файлы для UE-V, выполните одно из указанных ниже действий.

-   Перед развертыванием агента UE-V установите флажок "не использовать автономные файлы" в параметрах групповой политики UE-V.

-   Во время установки UE-V задайте параметр AgentSetup.exe в `SyncMethod = None` командной строке или пакетном файле. Дополнительные сведения о развертывании агента можно найти в разделе [развертывание агента UE-V](deploying-the-ue-v-agent.md).

Если отключить параметр автономных файлов для UE-V и вы не указали параметр **PowerSchool** во время установки, установка агента UE-V завершается сбоем. Вы также можете отключить автономные файлы с помощью PowerShell или WMI. Дополнительные сведения о командах WMI и PowerShell можно найти в разделе [Управление агентом UE-V 1,0 и пакетами с помощью PowerShell и WMI](managing-the-ue-v-10-agent-and-packages-with-powershell-and-wmi.md).

Чтобы начать синхронизацию параметров, необходимо перезагрузить компьютер.

### Предварительные требования для компонента UE-V PowerShell

Для агента UE-V PowerShell требуется платформа .NET Framework версии 3,5 с пакетом обновления 1 (SP1) и PowerShell версии 2,0 или выше.

### Необходимые условия для поддержки генераторов UE-V

Установите генератор UE-V на компьютере, который используется для создания настраиваемых шаблонов расположений параметров. На этом компьютере должны быть установлены приложения, параметры которых будут перемещаться. Вы должны быть участником группы "Администраторы" на компьютере, на котором запущено программное обеспечение для создания генератора UE-V. Кроме того, на компьютере, на котором используется файловая система NTFS, должен быть установлен генератор UE-V. Для использования программного обеспечения генератора UE-V требуется платформа .NET Framework версии 4. Дополнительные сведения можно найти в разделе [Планирование развертывания настраиваемого шаблона для UE-V 1,0](planning-for-custom-template-deployment-for-ue-v-10.md).
