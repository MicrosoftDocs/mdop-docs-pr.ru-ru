## <span data-ttu-id="968a5-123">Известные проблемы с РАСШИРЕНными разрешениями см 4,0</span><span class="sxs-lookup"><span data-stu-id="968a5-123">Known issues with AGPM 4.0</span></span>


### <span data-ttu-id="968a5-124">Команда "Импорт из производства" не импортирует параметры в возвращенный объект групповой политики</span><span class="sxs-lookup"><span data-stu-id="968a5-124">Import from Production command does not import settings into a GPO that is checked out</span></span>

<span data-ttu-id="968a5-125">Если вы изменяете объект групповой политики в производственной среде, вы должны импортировать объект групповой политики из производства, чтобы обновить GPO в автономном архиве.</span><span class="sxs-lookup"><span data-stu-id="968a5-125">If you edit a GPO in the production environment, you must import the GPO from production to update the GPO in the offline archive.</span></span> <span data-ttu-id="968a5-126">Команда " **Импорт из производства** " предназначена для выполнения финальной резервной копии перед завершением редактирования, чтобы при необходимости вернуться к рабочей резервной копии.</span><span class="sxs-lookup"><span data-stu-id="968a5-126">The **Import from Production** command is intended to let you perform a final production backup before you finish editing so that you can roll back to the production backup if it is necessary.</span></span>

<span data-ttu-id="968a5-127">Если объект групповой политики извлекается при выполнении команды **Импорт из производства** , изменения, внесенные в нее, не вносятся в извлеченную версию объекта групповой политики.</span><span class="sxs-lookup"><span data-stu-id="968a5-127">If the GPO is checked out when you run the **Import from Production** command, then the production changes are not incorporated into the checked out version of the GPO.</span></span> <span data-ttu-id="968a5-128">Однако импортированная версия объекта групповой политики добавляется в журнал объекта групповой политики, несмотря на то, что эта версия недоступна для редактирования.</span><span class="sxs-lookup"><span data-stu-id="968a5-128">However, the imported version of the GPO is added to the history of the GPO even though that version is not available to be edited.</span></span> <span data-ttu-id="968a5-129">Когда объект групповой политики возвращен, эта версия заменяет импортированную версию в архиве, но обе эти версии будут доступны в истории объекта групповой политики.</span><span class="sxs-lookup"><span data-stu-id="968a5-129">When the GPO is checked in, that version will supersede the imported version in the archive, but both are available in the history of the GPO.</span></span>

<span data-ttu-id="968a5-130">**Временное решение:** Убедитесь, что объект групповой политики возвращен перед импортом из рабочей области.</span><span class="sxs-lookup"><span data-stu-id="968a5-130">**Workaround:** Make sure that the GPO is checked in before you import it from production.</span></span> <span data-ttu-id="968a5-131">Если объект групповой политики не был возвращен перед импортом, вы можете отменить изменения и вернуться к версии объекта групповой политики, импортированной из производства, с помощью команды " **Отменить извлечение** ".</span><span class="sxs-lookup"><span data-stu-id="968a5-131">If the GPO was not checked in before you imported it, you can use the **Undo Check Out** command to discard your changes and roll back to the version of the GPO that you imported from production.</span></span>

### <span data-ttu-id="968a5-132">Извлеченные объекты групповой политики нельзя редактировать в течение нескольких минут в среде, использующей топологию Active Directory с несколькими сайтами.</span><span class="sxs-lookup"><span data-stu-id="968a5-132">Checked out GPOs cannot be edited for several minutes in an environment that uses a multiple site Active Directory topology</span></span>

<span data-ttu-id="968a5-133">В расширенном управлении групповыми политиками используется модель клиент-сервер.</span><span class="sxs-lookup"><span data-stu-id="968a5-133">AGPM uses a client/server model.</span></span> <span data-ttu-id="968a5-134">Сервер управления групповыми политиками и клиент расширенного управления групповыми политиками определяют собственный ближайший контроллер домена для операций групповой политики.</span><span class="sxs-lookup"><span data-stu-id="968a5-134">The AGPM Server and the AGPM Client each determine their own closest domain controller for Group Policy operations.</span></span> <span data-ttu-id="968a5-135">При извлечении объекта групповой политики с помощью клиента с расширенным управлением групповыми политиками на самом деле это сервер политики РАСШИРЕНного поиска, который извлекает их из автономного архива во временную папку в папке SYSVOL.</span><span class="sxs-lookup"><span data-stu-id="968a5-135">When you check out a GPO by using an AGPM Client, it is actually the AGPM Server that checks the GPO out from the offline archive to a temporary folder in the SYSVOL folder.</span></span>

<span data-ttu-id="968a5-136">Если сервер и клиент расширенного управления групповыми политиками находятся на разных сайтах, временный извлеченный объект GPO может отсутствовать на контроллере домена локального сайта в течение нескольких минут или до 30 минут из-за задержки репликации SYSVOL.</span><span class="sxs-lookup"><span data-stu-id="968a5-136">If the AGPM Server and the AGPM Client are in different sites, then the temporary checked out GPO may not be present on the local site's domain controller for several minutes or up to 30 minutes due to SYSVOL replication latency.</span></span> <span data-ttu-id="968a5-137">В этой ситуации невозможно изменить извлеченный объект групповой политики с помощью консоли управления групповыми политиками в клиенте с расширенными политиками, пока не будет выполнена репликация SYSVOL для извлеченного объекта групповой политики.</span><span class="sxs-lookup"><span data-stu-id="968a5-137">In this situation, you cannot edit the checked out GPO using the GPMC on an AGPM Client until SYSVOL replication of the checked out GPO has occurred.</span></span>

<span data-ttu-id="968a5-138">**Временное решение:** Рекомендуется расположить клиенты с РАСШИРЕНным контролем учетных изменений на том же сайте, что и сервер с РАСШИРЕНными правами, к которому они подключаются, чтобы не дождаться завершения репликации SYSVOL, прежде чем можно будет изменить извлеченный объект групповой политики.</span><span class="sxs-lookup"><span data-stu-id="968a5-138">**Workaround:** As a best practice, you should position AGPM Clients in the same site as the AGPM Server to which they connect so that you do not have to wait for SYSVOL replication to occur before you can edit a checked out GPO.</span></span>

### <span data-ttu-id="968a5-139">Не удается прочитать предельный размер резервной копии, если у вашей учетной записи нет разрешений на доступ к архиву</span><span class="sxs-lookup"><span data-stu-id="968a5-139">AGPM cannot read the backup limit if your account does not have permissions for the archive</span></span>
