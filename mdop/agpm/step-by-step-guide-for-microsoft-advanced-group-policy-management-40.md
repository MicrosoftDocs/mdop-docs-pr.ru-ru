# Пошаговое руководство по средству расширенного управления групповыми политиками Майкрософт4.0


В этом пошаговом руководстве показаны сложные методики для управления групповыми политиками, которые используют консоль управления групповыми политиками (GPMC) и расширенные возможности управления групповыми политиками (Майкрософт). Функция расширенного управления групповыми политиками расширяет возможности GPMC, обеспечивая:

-   Стандартные роли для делегирования разрешений на управление объектами групповой политики (GPO) нескольким администраторам групповых политик в дополнение к возможности делегирования доступа к GPO в рабочей среде.

-   Архив, позволяющий администраторам групповых политик создавать и изменять GPO в автономном режиме перед развертыванием объектов GPO в рабочей среде.

-   Возможность вернуться к предыдущей версии GPO и ограничить количество версий, сохраненных в архиве.

-   Возможность возврата и извлечения объектов GPO, чтобы администраторы групповых политик не случайно перезаписывали работу друг друга.

-   Возможность поиска объектов групповой политики с определенными атрибутами и фильтрации списка объектов групповой политики, отображаемых в списке.

## Общие сведения о сценарии РАСШИРЕНного использования функций


В этом сценарии для каждой роли в разделе управления ГРУППОВыми политиками используется отдельная учетная запись, чтобы показать, как можно управлять групповой политикой в среде с несколькими администраторами групповой политики с разными уровнями разрешений. В частности, вы будете выполнять следующие задачи:

-   Используя учетную запись, которая входит в группу "Администраторы домена", установите сервер доменных групп и назначьте роль администратора РАСШИРЕНной учетной записи или группе.

-   Используя учетные записи, которым будут назначаться роли для РАСШИРЕНного использования функций, установите Клиентская политика

-   С помощью учетной записи, которая имеет роль администратора РАСШИРЕНного доступа к данным, настройте для них доступ к объектам групповой политики, назначая роли другим учетным записям.

-   Из учетной записи, в которой находится роль редактора, необходимо запросить создание нового объекта групповой политики, который затем утверждается с помощью учетной записи, которая имеет роль утверждающего. С помощью учетной записи редактора извлеките GPO из архива, измените объект групповой политики, проверьте GPO в архиве и запросите развертывание.

-   С помощью учетной записи, которая имеет роль утверждающего, проверьте объект групповой политики и разверните его в рабочей среде.

-   С помощью учетной записи, в которой находится роль редактора, создайте шаблон GPO и используйте его в качестве отправной точки для создания нового объекта групповой политики.

-   С помощью учетной записи, в которой находится роль утверждающего, удалите и восстановите объект групповой политики.

### Требования к серверу для РАСШИРЕНного сервера

Для работы сервера расширенного управления групповыми политиками требуется Windows Server2008R2, Windows Server2008, Windows7 и GPMC из средств удаленного администрирования сервера (RSAT) или Windows Vista с установленным пакетом обновления 1 (SP1) и GPMC из RSAT. Поддерживаются как 32, так и 64-разрядные версии.

Перед установкой сервера РАСШИРЕНного доменных прав необходимо быть членом группы администраторов домена и следующими функциями Windows, если не указано иное.

-   ПОЛИТИКОЙ

    -   Windows Server2008R2 или Windows Server2008: Если консоль GPMC не отображается, она будет автоматически установлена с помощью расширенного управления групповыми политиками.

    -   Windows7: перед установкой многоэлементного управления групповыми политиками необходимо установить GPMC из RSAT. Дополнительные сведения можно найти в разделе [средства администрирования сервера удаленного доступа для Windows 7](https://go.microsoft.com/fwlink/?LinkID=131280) ( https://go.microsoft.com/fwlink/?LinkID=131280) .

    -   Windows Vista с пакетом обновления 1 (SP1): перед установкой расширенного управления групповыми политиками необходимо установить GPMC из RSAT. Дополнительные сведения можно найти [в разделе средства удаленного администрирования сервера для Windows Vista с пакетом обновления 1](https://go.microsoft.com/fwlink/?LinkID=116179) (SP1) ( https://go.microsoft.com/fwlink/?LinkID=116179) .

-   Платформа .NET Framework 3,5 или более поздней версии

    -   Windows Server2008R2 или Windows7: Если версия .NET Framework 3,5 или более поздней версии отсутствует, .NET Framework 3,5 автоматически устанавливается с помощью расширенного управления групповыми политиками.

    -   Windows Server2008 или Windows Vista с пакетом обновления 1 (SP1): перед установкой расширенного управления групповыми политиками необходимо установить .NET Framework 3,5 или более поздней версии.

Следующие компоненты Windows необходимы для сервера РАСШИРЕНного отслеживания и автоматически устанавливаются, если они отсутствуют.

-   Активация WCF; Активация не по протоколу HTTP

-   Служба активации процессов Windows

    -   Модель процесса

    -   Среда .NET

    -   API конфигурации

### Требования к сценарию

Перед тем как приступить к созданию этого сценария, создайте четыре учетных записи пользователей. Во время сценария вы будете назначать одну из следующих ролей РАСШИРЕНного управления правами на доступ каждому из этих учетных записей: администратор РАСШИРЕНного доступа (полный доступ), утверждающий, редактор и рецензент. Эти учетные записи должны быть доступны для отправки и получения сообщений электронной почты. Назначьте **объектам GPO** разрешение на доступ к учетным записям, имеющим администратора, утверждающего и (необязательно) РОЛЕЙ редактора расширенного доступа.

**Примечание** 
 Разрешение " **связывание объектов GPO** " назначается членам группы администраторов домена и администраторов предприятия по умолчанию. Чтобы назначить разрешение на доступ к **объектам GPO** для дополнительных пользователей или групп (например, для учетных записей, которые обладают ролями администратора или утверждающего), щелкните узел **домена, а**затем откройте вкладку **Делегирование** , нажмите кнопку **Добавить**, а затем выберите пункт Пользователи или группы, которым нужно назначить разрешение.

### <a href="" id="bkmk-config1"></a>Шаг 1: Установка сервера РАСШИРЕНного набора элементов

На этом этапе необходимо установить сервер РАСШИРЕНного управления правами на рядовом сервере или контроллере домена, который будет запускать службу расширенного управления групповыми политиками, и настроить этот архив. Все операции РАСШИРЕНного управления правами обрабатываются в этой службе Windows и выполняются с учетными данными службы. Архивация, управляемая сервером расширенного управления групповыми политиками, может быть размещена на этом сервере или на другом сервере в том же лесе.

**Установка сервера расширенного управления групповыми политиками на компьютере, на котором будет размещена служба РАСШИРЕНного управления**

1.  Войдите в систему под учетной записью, которая входит в группу "Администраторы домена".

2.  Запустите компакт-диск с пакетом оптимизации рабочей среды Microsoft и следуйте инструкциям на экране, чтобы выбрать **Расширенное управление групповыми политиками — сервер**.

3.  В диалоговом окне **Добро пожаловать** нажмите кнопку **Далее**.

4.  В диалоговом окне **условия лицензионного соглашения на использование программного обеспечения Microsoft** подтвердите условия и нажмите кнопку **Далее**.

5.  В диалоговом окне **путь к приложению** выберите расположение, в которое нужно установить сервер расширенного набора. На компьютере, на котором установлен сервер расширенного управления групповыми политиками, будет размещена служба и Управление архивом. Нажмите кнопку **Далее**.

6.  В диалоговом окне " **архивный путь** " выберите расположение для архива относительно сервера расширенного поиска. Путь к архиву может указывать на папку на сервере РАСШИРЕНного поиска и в других местах. Тем не менее, необходимо выбрать расположение, в котором достаточно свободного места для хранения всех объектов групповой политики и данных журнала, управляемых этим сервером управления групповыми политиками. Нажмите кнопку **Далее**.

7.  В диалоговом окне **учетная запись службы расширенного** выбора параметров выберите учетную запись службы, под которой будет выполняться служба расширенного выбора и нажмите кнопку **Далее**.

    Эта учетная запись должна входить в группу "Администраторы домена" или, в случае наименее привилегий, в следующих группах каждого домена, управляемых сервером управления групповыми политиками:

    -   Владельцы создателей групповой политики

    -   Операторы архива

    Кроме того, для этой учетной записи требуется разрешение полного доступа для следующих папок:

    -   Папка архива РАСШИРЕНного доступа к данным, для которой это разрешение автоматически предоставляется во время установки сервера с РАСШИРЕНными правами, если он установлен на локальном диске.

    -   Папка Local System Temp, обычно%windir%\\Temp.

8.  В диалоговом окне **Владелец архива** выберите учетную запись или группу, которой назначена роль администратора расширенного управления правами (полный доступ). Администраторы РАСШИРЕНного доступа к учетным группам могут назначать роли и разрешения РАСШИРЕНных групповых политик другим администраторам групповой политики, чтобы позже можно было назначить роль администратора с помощью функции РАСШИРЕНного администрирования дополнительным администраторам групповых политик. Для этого сценария выберите учетную запись, которая будет использоваться в роли администратора РАСШИРЕНного администрирования. Нажмите кнопку **Далее**.

9.  В диалоговом окне **Настройка порта** введите порт для прослушивания службой расширенного выбора ключа. Не снимайте флажок **Добавить исключение для порта в брандмауэр** , если только вы не настраиваете исключения для порта вручную или не используйте правила для настройки исключений. Нажмите кнопку **Далее**.

10. В диалоговом окне **языки** выберите один или несколько языков интерфейса, которые нужно установить для сервера с расширенным интерфейсом.

11. Нажмите кнопку **установить**, а затем — **Готово** , чтобы завершить работу мастера установки.

    **Внимание!**  Не изменяйте параметры службы РАСШИРЕНного **управления правами с помощью средств администрирования** и **служб** в операционной системе. Это может привести к невозможности запуска службы РАСШИРЕНного рассчета. Сведения о том, как изменить параметры службы, приведены в разделе Справка по расширенному управлению групповыми политиками.

     

### <a href="" id="bkmk-config2"></a>Шаг 2. Установка клиента с расширенным управлением групповыми политиками

Каждый администратор групповой политики — любой пользователь, который создает, редактирует, развертывает или удаляет GPO, должен установить на компьютеры, которые они используют, для управления объектами групповой политики, клиент с РАСШИРЕНными правами. Узел контрольных изменений, который вы используете для выполнения многих задач управления GPO, отображается в консоли управления групповыми политиками только в том случае, если вы установили клиент РАСШИРЕНного управления правами. Для этого сценария необходимо установить клиентская версия с РАСШИРЕНным Управлением по крайней мере на одном компьютере. На компьютерах конечных пользователей, которые не выполняют администрирование групповой политики, вам не нужно устанавливать клиент с расширенным управлением групповыми политиками.

**Установка клиента с расширенным управлением групповыми политиками на компьютере администратора групповой политики**

1.  Запустите компакт-диск с пакетом оптимизации рабочей среды Microsoft и следуйте инструкциям на экране, чтобы выбрать **Расширенное управление групповыми политиками — клиент**.

2.  В диалоговом окне **Добро пожаловать** нажмите кнопку **Далее**.

3.  В диалоговом окне **условия лицензионного соглашения на использование программного обеспечения Microsoft** подтвердите условия и нажмите кнопку **Далее**.

4.  В диалоговом окне **путь к приложению** выберите расположение для установки клиента с расширенным управлением групповыми политиками. Нажмите кнопку **Далее**.

5.  В диалоговом окне **сервер расширенного выбора сервера** введите DNS-имя или IP-адрес сервера расширенного выбора и порта, к которому вы хотите подключиться. Портом по умолчанию для службы РАСШИРЕНного обслуживания не является 4600. Не снимайте флажок **Разрешить использование консоли управления Майкрософт через брандмауэр** , если только вы не настраиваете исключения для порта вручную или не используйте правила для настройки исключений. Нажмите кнопку **Далее**.

6.  В диалоговом окне **языки** выберите один или несколько языков интерфейса, которые нужно установить для клиента с расширенным управлением групповыми политиками.

7.  Нажмите кнопку **установить**, а затем — **Готово** , чтобы завершить работу мастера установки.

### <a href="" id="bkmk-config3"></a>Шаг 3: Настройка подключения к серверу с РАСШИРЕНным интерфейсом

Управление ГРУППОВыми политиками сохраняет все версии каждого управляемого объекта групповой политики (GPO), то есть каждый GPO, для которого в РАСШИРЕНном архиве находятся элементы управления изменениями. Это позволяет администраторам групповых политик просматривать и изменять GPO в автономном режиме без немедленного воздействия на развернутую версию каждого объекта групповой политики.

На этом этапе необходимо настроить соединение с сервером РАСШИРЕНного администрирования и убедиться, что все администраторы групповой политики подключаются к одному и тому же серверу. (Сведения о настройке нескольких серверов расширенного управления групповыми политиками см. в разделе Справка по расширенному управлению групповой политикой.)

**Настройка подключения к серверу РАСШИРЕНного конфигурирования для всех администраторов групповой политики**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которую выбрали в качестве владельца архива. У этого пользователя есть роль администратора управления групповыми политиками (полный доступ).

2.  Нажмите кнопку **Пуск**, выберите пункт **Администрирование**, а затем — **Управление групповой политикой** , чтобы открыть консоль управления групповыми политиками.

3.  Изменение объекта групповой политики, применяемого ко всем администраторам групповой политики.

4.  В окне **редактора управления групповыми политиками** дважды щелкните элемент **Конфигурация пользователя**, **политики**, **Административные шаблоны**, **компоненты Windows**и управление **групповыми**политиками.

5.  В области сведений дважды щелкните элемент расширенной настройки и выберите **сервер по умолчанию (все домены)**.

6.  В окне **Свойства** выберите параметр **включена** и введите DNS-имя или IP-адрес и порт (например, **Server.contoso.com:4600**) для сервера, на котором размещается Архив. По умолчанию служба РАСШИРЕНного использования порта использует порт 4600.

7.  Нажмите кнопку **ОК**, а затем закройте окно **редактора управления групповыми политиками** . После обновления групповой политики соединение с сервером РАСШИРЕНного политики для каждого администратора будет настроено.

### <a href="" id="bkmk-config4"></a>Шаг 4: Настройка уведомления по электронной почте

Как Администратор расширенного управления групповыми политиками (полный доступ) вы указываете адреса электронной почты утверждающих и администраторов РАСШИРЕНного доступа, которым будет отправлено сообщение электронной почты с запросом, когда редактор попытается создать, развернуть или удалить объект групповой политики. Кроме того, вы можете определить псевдоним, из которого отправляются эти сообщения.

**Настройка уведомления по электронной почте для РАСШИРЕНного**

1.  В **редакторе управления групповыми политиками** перейдите к папке " **Управление изменениями** ". 

2.  В области сведений откройте вкладку **Делегирование домена** .

3.  В поле **из адреса электронной почты** введите псевдоним электронной почты для расширенного вида, из которого нужно отправлять уведомления.

4.  В поле **адрес электронной почты** введите адрес электронной почты учетной записи пользователя, которому вы хотите назначить роль утверждающего.

5.  В поле **SMTP-сервер** введите допустимый почтовый SMTP-сервер.

6.  В полях **имя пользователя** и **пароль** введите учетные данные пользователя, у которого есть доступ к службе SMTP. Нажмите кнопку **Применить**.

### <a href="" id="bkmk-config5"></a>Шаг 5: передача прав доступа

Как Администратор расширенного управления групповыми политиками (режим полного доступа) вы делегируйте доступ к объектам GPO на уровне домена, назначая роли учетной записи каждого администратора групповой политики.

**Примечание**  Вы также можете делегировать доступ на уровне объекта групповой политики, а не на уровне домена. Дополнительные сведения можно найти в разделе Справка по расширенному управлению групповыми политиками.

 

**Важно!**  Вы должны ограничить членство в группе Владельцы создателей групповой политики, чтобы не использовать управление правами на доступ к объектам GPO. (В **консоли управления групповыми политиками**щелкните **объекты групповой политики** в лесу и домене, в которых вы хотите управлять объектами GPO, выберите пункт **Делегирование**и настройте параметры в соответствии с потребностями Организации.)

 

**Передача доступа ко всем объектам групповой политики внутри домена**

1.  На вкладке **доменное делегирование** нажмите кнопку **Добавить** , выберите учетную запись администратора групповой политики, которая будет использоваться в качестве утверждающего, и нажмите кнопку **ОК**.

2.  В диалоговом окне **Добавление группы или пользователя** выберите роль **утверждающего** , чтобы назначить ей роль учетной записи, а затем нажмите кнопку **ОК**. (Эта роль включает в себя роль проверяющего.)

3.  Нажмите кнопку **Добавить** , выберите учетную запись пользователя администратора групповой политики, которая будет использоваться в качестве редактора, и нажмите кнопку **ОК**.

4.  В диалоговом окне **Добавление группы или пользователя** выберите роль " **Редактор** ", чтобы назначить эту роль учетной записи, а затем нажмите кнопку **ОК**. (Эта роль включает в себя роль проверяющего.)

5.  Нажмите кнопку **Добавить** , выберите учетную запись администратора групповой политики, которая будет использоваться в качестве рецензента, и нажмите кнопку **ОК**.

6.  В диалоговом окне **Добавление группы или пользователя** выберите роль **проверяющего** , чтобы назначить учетной записи только эту роль.

### <a href="" id="bkmk-manage1"></a>Действие 1: создание объекта групповой политики

В среде с несколькими администраторами групповой политики пользователи, которым назначена роль редактора, могут запросить создание новых GPO. Тем не менее, этот запрос должен быть утвержден участником с ролью утверждающего.

На этом этапе вы используете учетную запись с ролью редактора, чтобы запросить создание нового объекта групповой политики. С помощью учетной записи, которая имеет роль утверждающего, вы утверждаете этот запрос для создания объекта групповой политики.

**Запрос создания нового объекта групповой политики с помощью расширенного управления групповыми политиками**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которой назначена роль "редактор".

2.  В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

3.  Щелкните правой кнопкой мыши узел **контроль изменений** и выберите команду **создать управляемый объект групповой политики**.

4.  В диалоговом окне **новый управляемый объект групповой политики** :

    1.  Чтобы получить копию запроса, введите свой адрес электронной почты в поле **"копия"** .

    2.  Введите **MyGPO** в качестве имени нового объекта групповой политики.

    3.  Введите Примечание для нового объекта групповой политики.

    4.  Нажмите кнопку **создать в реальном времени** , чтобы новый объект групповой политики развернут в рабочей среде сразу после утверждения. Нажмите кнопку **Отправить**.

5.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Новый объект групповой политики появится на вкладке **ожидающие** .

**Утверждение ожидающего запроса на создание объекта групповой политики**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которая имеет роль утверждающего в РАСШИРЕНном.

2.  Откройте папку "Входящие" электронной почты для учетной записи и обратите внимание на то, что вы получили сообщение электронной почты от псевдонима с помощью запроса редактора, чтобы создать объект групповой политики.

3.  В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

4.  На вкладке **содержание** откройте вкладку **ожидается** , чтобы отобразить ожидающие GPO.

5.  Щелкните правой кнопкой мыши **MyGPO**и выберите команду **утвердить**.

6.  Нажмите кнопку **Да** , чтобы подтвердить утверждение, и переместите объект групповой политики на вкладку **Управление** .

### <a href="" id="bkmk-manage2"></a>Действие 2: изменение объекта групповой политики

С помощью объектов групповой политики можно настраивать параметры компьютера или пользователя и развертывать их на нескольких компьютерах или в других пользователей. На этом этапе вы используете учетную запись с ролью "редактор" для извлечения объекта групповой политики из архива, изменения объекта групповой политики в автономном режиме, проверки редактируемого GPO в архиве и запроса развертывания объекта групповой политики в рабочей среде. В этом сценарии вы можете настроить параметр в объекте групповой политики таким образом, чтобы пароль имел длину не менее восьми символов.

**Извлечение GPO из архива для редактирования**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которая имеет роль редактора в РАСШИРЕНном наборе элементов.

2.  В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

3.  На вкладке **содержание** в области сведений откройте вкладку **Управление** , чтобы отобразить управляемые объекты групповой политики.

4.  Щелкните правой кнопкой мыши **MyGPO**и выберите пункт **извлечь.**

5.  Введите Примечание, которое будет отображаться в истории объекта групповой политики, когда он извлечен, и нажмите кнопку **ОК**.

6.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. На вкладке **Управление** состоянием GPO считается **извлеченным**.

**Изменение GPO в автономном режиме и Настройка минимальной длины пароля**

1.  На вкладке **управляемые** щелкните правой кнопкой мыши **MyGPO**и выберите команду **изменить** , чтобы открыть окно **редактора управления групповыми политиками** и изменить автономную копию объекта групповой политики. Для этого сценария настройте минимальную длину пароля:

    1.  В разделе **Конфигурация компьютера**дважды щелкните **политики**, **Параметры Windows**, **Параметры безопасности**, **политики учетных записей**и **политики паролей**.

    2.  В области сведений дважды щелкните элемент **Минимальная длина пароля**.

    3.  В окне "Свойства" установите флажок **определить этот параметр политики** , укажите число символов **8**и нажмите кнопку **ОК**.

2.  Закройте окно **редактора управления групповыми политиками** .

**Проверка GPO в архиве**

1.  На вкладке **Управление** , щелкните правой кнопкой мыши **MyGPO** и выберите команду **вернуть**.

2.  Введите примечание и нажмите кнопку **ОК**.

3.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. На вкладке **Управление** состоянием объекта групповой политики считается **возвращенное**.

**Запрос развертывания объекта групповой политики в рабочей среде**

1.  На вкладке **Управление** , щелкните правой кнопкой мыши **MyGPO** и выберите команду **развернуть**.

2.  Поскольку эта учетная запись не является администратором утверждения или РАСШИРЕНного администрирования, необходимо отправить запрос на развертывание. Чтобы получить копию запроса, введите свой адрес электронной почты в поле **"копия"** . Введите Примечание, которое будет отображаться в журнале объекта групповой политики, а затем нажмите кнопку **Отправить**.

3.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. **MyGPO** отображается в списке объектов групповой политики на вкладке **ожидающие** .

### <a href="" id="bkmk-manage3"></a>Шаг 3: Проверка и развертывание объекта групповой политики

На этом этапе вы действуете как утверждающий, создавая отчеты и анализируя параметры и изменяя параметры в объекте групповой политики, чтобы определить, нужно ли их утверждать. После того как вы оцените объект групповой политики, вы развернете его в производственную среду и свяжите объект групповой политики с доменом или подразделением (OU). GPO вступает в силу, когда групповая политика обновляется на компьютерах в этом домене или OU.

**Просмотр параметров в объекте групповой политики**

1. На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему под учетной записью пользователя, которому назначена роль утверждающего в расширенном управлении правами. Любой администратор групповой политики с ролью проверяющего, который входит в состав других ролей, может просматривать параметры в объекте групповой политики.

2. Откройте папку "Входящие" электронной почты для учетной записи и обратите внимание на то, что вы получили сообщение электронной почты от псевдонима РАСШИРЕНного использования запроса редактора для развертывания объекта групповой политики.

3. В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

4. На вкладке **содержание** в области сведений откройте вкладку **ожидается** .

5. Дважды щелкните **MyGPO** , чтобы отобразить историю.

6. Проверьте параметры в последней версии MyGPO:

   1.  В окне " **Журнал** " щелкните правой кнопкой мыши версию объекта групповой политики с последней меткой времени, выберите пункт **Параметры**, а затем — **отчет HTML** , чтобы отобразить сводку по параметрам объекта групповой политики.

   2.  В веб-браузере нажмите кнопку **Показать все** , чтобы отобразить все параметры в объекте групповой политики. Закройте браузер.

7. Сравните последнюю версию MyGPO с первой версией, возвращенной в Архив:

   1. В окне " **Журнал** " выберите версию объекта групповой политики с последней меткой времени. Нажмите клавишу CTRL, а затем выберите самую старую версию **GPO, которой не является *** * \ \ * * *.

   2. Нажмите кнопку **различия** . Раздел **политики учетных записей и политика паролей** выделены зеленым цветом и предшествует " **\ [+ \]**. Это указывает на то, что параметр настроен только в последней версии GPO.

   3. Выберите политики **учетных записей и политика паролей**. Параметр **минимальной длины пароля** также выделяется зеленым цветом и предшествует **\ [+ \]**, что указывает на то, что оно настроено только в последней версии GPO.

   4. Закройте веб-браузер.

**Развертывание объекта групповой политики в рабочей среде**

1.  На вкладке **Ожидание** щелкните правой кнопкой мыши **MyGPO** и выберите команду **утвердить**.

2.  Введите Примечание для включения в историю объекта групповой политики.

3.  Щелкните **Да**. После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Объект групповой политики развернут в рабочей среде.

**Связывание объекта групповой политики с доменом или подразделением**

1.  В консоли управления групповыми политиками щелкните правой кнопкой мыши домен или подразделение (OU), к которому вы хотите применить настраиваемый объект групповой политики, а затем выберите команду **связать существующий объект групповой политики**.

2.  В диалоговом окне **Выбор объекта групповой политики** щелкните **MyGPO**, а затем нажмите кнопку **ОК**.

### <a href="" id="bkmk-manage4"></a>Шаг 4: использование шаблона для создания объекта групповой политики

На этом этапе вы используете учетную запись с ролью "редактор" для создания и использования шаблона. Этот шаблон — это статическая версия GPO, которая используется в качестве отправной точки для создания новых объектов групповой политики. Несмотря на то, что вы не можете изменять шаблон, вы можете создать новый объект групповой политики на основе шаблона. Шаблоны полезны для быстрого создания нескольких GPO, включающих в себя множество одинаковых параметров политики.

**Создание шаблона на основе существующего объекта групповой политики**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которой назначена роль редактора в РАСШИРЕНном наборе данных.

2.  В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

3.  На вкладке **содержание** в области сведений откройте вкладку **Управление** .

4.  Щелкните правой кнопкой мыши **MyGPO**и выберите команду **Сохранить как шаблон** , чтобы создать шаблон, включающий все параметры, в настоящее время находящиеся в MyGPO.

5.  Введите **MyTemplate** в качестве имени шаблона и примечания, а затем нажмите кнопку **ОК**.

6.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Новый шаблон появится на вкладке **шаблоны** .

**Запрос создания нового объекта групповой политики с помощью расширенного управления групповыми политиками**

1.  Откройте вкладку **Управление** .

2.  Щелкните правой кнопкой мыши узел **контроль изменений** и выберите команду **создать управляемый объект групповой политики**.

3.  В диалоговом окне **новый управляемый объект групповой политики** :

    1.  Чтобы получить копию запроса, введите свой адрес электронной почты в поле **"копия"** .

    2.  Введите **MyOtherGPO** в качестве имени нового объекта групповой политики.

    3.  Введите Примечание для нового объекта групповой политики.

    4.  Нажмите кнопку **создать в реальном времени** , чтобы новый объект групповой политики развернут в рабочей среде сразу после утверждения.

    5.  **В шаблоне из GPO**выберите **MyTemplate**. Нажмите кнопку **Отправить**.

4.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Новый объект групповой политики появится на вкладке **ожидающие** .

Используйте учетную запись, которой назначена роль утверждающего, чтобы утвердить ожидающий запрос на создание объекта групповой политики, как на [этапе 1: Создайте объект групповой политики](#bkmk-manage1). MyTemplate включает все параметры, настроенные в MyGPO. Поскольку MyOtherGPO был создан с помощью MyTemplate, он сначала содержит все параметры, MyGPO содержащиеся на момент создания MyTemplate. Чтобы подтвердить это, вы можете создать отчет о разнице, чтобы сравнить MyOtherGPO с MyTemplate.

**Извлечение GPO из архива для редактирования**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему с помощью учетной записи пользователя, которой назначена роль редактора в РАСШИРЕНном наборе данных.

2.  Щелкните правой кнопкой мыши **MyOtherGPO**и выберите пункт **извлечь.**

3.  Введите Примечание, которое будет отображаться в истории объекта групповой политики, когда он извлечен, и нажмите кнопку **ОК**.

4.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. На вкладке **Управление** состоянием GPO считается **извлеченным**.

**Изменение GPO в автономном режиме и Настройка длительности блокировки учетной записи**

1.  На вкладке **управляемые** щелкните правой кнопкой мыши **MyOtherGPO**и выберите команду **изменить** , чтобы открыть окно **редактора управления групповыми политиками** и изменить автономную копию объекта групповой политики. Для этого сценария настройте минимальную длину пароля:

    1.  В разделе **Конфигурация компьютера**дважды щелкните **политики**, **Параметры Windows**, **Параметры безопасности**, **политики учетных записей**и **политики блокировки учетных записей**.

    2.  В области сведений дважды щелкните **Продолжительность блокировки учетной записи**.

    3.  В окне "Свойства" установите флажок **задать этот параметр политики**, задайте для параметра продолжительность **30** минут и нажмите кнопку **ОК**.

2.  Закройте окно **редактора управления групповыми политиками** .

Проверьте MyOtherGPO в архиве и запросите развертывание в том виде, в котором вы делали в MyGPO на [этапе 2. Измените объект групповой политики](#bkmk-manage2). Вы можете сравнить MyOtherGPO с MyGPO или с MyTemplate с помощью отчетов о различиях. Любая учетная запись, включающая в себя роль рецензента (Управление групповыми политиками \ [полный доступ \], утверждающая, редактор или рецензент), может создавать отчеты.

**Сравнение GPO с другим GPO и с шаблоном**

1.  Чтобы сравнить MyGPO и MyOtherGPO, выполните указанные ниже действия.

    1.  На вкладке **Управление** нажмите кнопку **MyGPO**. Нажмите клавишу CTRL, а затем — **MyOtherGPO**.

    2.  Щелкните правой кнопкой мыши **MyOtherGPO**, наведите указатель на пункт **отличия**и выберите пункт **отчет в формате HTML**.

2.  Чтобы сравнить MyOtherGPO и MyTemplate, выполните указанные ниже действия.

    1.  На вкладке **Управление** нажмите кнопку **MyOtherGPO**.

    2.  Щелкните правой кнопкой мыши **MyOtherGPO**, наведите указатель на пункт **отличия**и выберите пункт **шаблон**.

    3.  Выберите **MyTemplate** и **HTML-отчет**, а затем нажмите кнопку **ОК**.

### <a href="" id="bkmk-manage5"></a>Шаг 5: удаление и восстановление объекта групповой политики

На этом этапе вы действуете как утверждающий для удаления GPO.

**Удаление объекта групповой политики**

1.  На компьютере с установленным клиентским управлением групповыми политиками Войдите в систему под учетной записью пользователя, которому назначена роль утверждающего.

2.  В дереве **консоли управления групповыми политиками** нажмите кнопку **изменить элемент управления** в лесу и домене, в котором вы хотите управлять объектами групповой политики.

3.  На вкладке **содержание** откройте вкладку **Управление** , чтобы отобразить управляемые объекты групповой политики.

4.  Щелкните правой кнопкой мыши **MyGPO**и выберите команду **Удалить**. Выберите команду **удалить объект групповой политики из архива и производства** , чтобы удалить версию в архиве и развернутую версию объекта групповой политики в рабочей среде.

5.  Введите Примечание, которое будет отображаться в контрольной записи для объекта групповой политики, и нажмите кнопку **ОК**.

6.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Объект групповой политики удаляется с вкладки " **управляемые** " и отображается на вкладке " **Корзина** ", в которой ее можно восстановить или уничтожить.

Иногда, когда вы удаляете объект групповой политики, он может потребоваться. На этом этапе вы действуете как утверждающее, чтобы восстановить удаленный объект групповой политики.

**Восстановление удаленного объекта групповой политики**

1.  На вкладке " **содержимое** " щелкните вкладку " **Корзина** ", чтобы отобразить удаленные GPO.

2.  Щелкните правой кнопкой мыши **MyGPO**и выберите команду **восстановить**.

3.  Введите Примечание, которое будет отображаться в журнале объекта групповой политики, и нажмите кнопку **ОК**.

4.  После того как в окне **ход выполнения работы с расширенным** состоянием работы с пределами будут завершены, нажмите **Закрыть**. Объект групповой политики удаляется из вкладки " **Корзина** " и отображается на вкладке " **управляемые** ".

    **Примечание**  Восстановление GPO в архиве не приводит к его повторному развертыванию в рабочей среде. Чтобы вернуть GPO в производственную среду, разверните объект групповой политики, как на [шаге 3: Проверка и развертывание объекта групповой политики](#bkmk-manage3).

     

После изменения и развертывания объекта групповой политики, возможно, вы обнаружите, что последние изменения в объекте групповой политики приводят к возникновению проблемы. На этом этапе вы выполняете роль утверждающего для отката к более ранней версии объекта групповой политики. Вы можете вернуться к любой версии в истории объекта групповой политики. С помощью примечаний и подписей вы можете определить известные хорошие версии и внести определенные изменения.

**Чтобы вернуться к более ранней версии объекта групповой политики**

1.  На вкладке **содержание** откройте вкладку **Управление** , чтобы отобразить управляемые объекты групповой политики.

2.  Дважды щелкните **MyGPO** , чтобы отобразить историю.

3.  Щелкните правой кнопкой мыши версию, которую нужно развернуть, и выберите команду **развернуть**, а затем — **Да**.

4.  После того как окно **прогресса** показывает, что весь ход выполнения завершен, нажмите кнопку **Закрыть**. В окне **журнала** нажмите кнопку **Закрыть**.

    **Примечание**  Чтобы убедиться в том, что версия, которая была повторно установлена, предназначена для проверки, просмотрите отчет о различиях для двух версий. В окне **журнала** для объекта групповой политики выберите две версии, щелкните их правой кнопкой мыши, наведите указатель на пункт **разница**и выберите **отчет HTML** или **отчет XML**.
